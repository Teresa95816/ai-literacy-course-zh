1. Set up Claude Code for your organization 組織部署決策地圖

Claude Code 透過 managed settings 強制執行組織政策, 優先於本機開發者設定, 可由 admin console, MDM, 或磁碟上的檔案送達, 控制 Claude 能觸及哪些工具, 指令, 伺服器, 網路目的地.

五個部署決策

選擇 API 供應商
Teams/Enterprise（預設建議）, Console, Bedrock, Vertex AI, Foundry
設定送達方式
server-managed, plist/registry, 檔案式 managed settings
決定強制項目
權限規則, sandbox, MCP 限制, plugin 限制, hooks 限制, 模型限制

OpenTelemetry, Analytics dashboard, 成本追蹤

部署完成後, 讓開發者執行 /status 檢視 Setting sources 是否顯示 Enterprise managed settings 及其來源（remote, plist, HKLM, HKCU, file）.

3. Configure auto mode 設定 auto mode 分類器

auto mode 讓 Claude Code 略過例行權限提示, 改用分類器審查每次工具呼叫, 封鎖不可逆, 破壞性, 或指向環境外的動作.用 autoMode.environment 告知分類器哪些 repo, bucket, 網域屬於組織信任範圍.

規則覆寫優先順序

1 · hard_deny
2 · soft_deny
3 · allow
4 · 明確意圖

無條件安全邊界, 使用者意圖與 allow 例外都不適用.

破壞性動作, 可被使用者意圖或 allow 例外解除.

覆寫符合的 soft_deny 規則作為例外.

訊息直接明確描述該動作時, 覆寫剩餘的 soft block.

CLI 子指令

claude auto-mode defaults 列出內建規則, claude auto-mode config 顯示實際生效設定, claude auto-mode critique 檢視自訂規則是否含糊或多餘.加入 "$defaults" 字串可在保留內建規則同時加入自訂項目.

4. Control MCP server access for your organization 控管 MCP server 存取

管理者可限制組織內可連接哪些 MCP server, 從固定部署一組伺服器到完全停用 MCP.

選擇模式 :
固定部署
核准清單
僅拒絕清單

managed-mcp.json 部署固定一組伺服器, 使用者無法新增或使用其他伺服器, 空的 server map 可完全停用 MCP.此檔案無法透過 server-managed settings 送達, 需經 MDM 部署到系統路徑.

allowedMcpServers 依 serverUrl（支援萬用字元）或 serverCommand（精確比對）過濾, 搭配 allowManagedMcpServersOnly: true 鎖定只採用 managed 來源.

deniedMcpServers 封鎖已知不良伺服器, 其餘全部允許, 拒絕清單永遠合併所有設定來源.

注意

serverName 只做精確比對且非安全控制, 因為名稱可被使用者任意命名, 要落實強制應使用 serverCommand 或 serverUrl.

5. Configure server-managed settings 伺服器管理設定

讓組織 Owner 從 claude.ai 主控台 Admin Settings 集中設定 Claude Code, 不需裝置管理基礎架構.客戶端在使用者以組織 OAuth 或直接設定的 API key 驗證時自動抓取設定, 每小時輪詢更新一次.

需求
Teams 或 Enterprise 方案, Owner 角色, 存取 api.anthropic.com
支援內容
幾乎所有 settings.json 欄位, 含 hooks, 環境變數
安全核准
shell 指令, 自訂環境變數, hooks, CLAUDE.md 需使用者同意
不適用情境
Bedrock, Vertex AI, Foundry, 自訂 ANTHROPIC_BASE_URL

第三方供應商情境改用自架的 Claude apps gateway 提供對等的遠端設定送達.

Claude Code 支援標準代理伺服器環境變數 HTTPS_PROXY, HTTP_PROXY, NO_PROXY（不支援 SOCKS 代理）.預設信任內建 Mozilla CA 憑證與作業系統憑證存放區, CLAUDE_CODE_CERT_STORE 可調整此行為.

注意

9. Run Claude Code through a gateway gateway 總覽

gateway 是組織自架, 位於 Claude Code 與模型供應商之間的代理, 由 gateway 用組織持有的憑證轉發請求.開發者以 gateway 核發的憑證驗證, 驗證, 用量追蹤, 預算, 稽核紀錄因此集中在一處.

兩種憑證

開發者憑證, 各自持有, 用於身分識別

供應商憑證, gateway 持有, 所有轉發流量共用

與訂閱的關係

設定 gateway 憑證時, 以 API 費率計入組織帳戶

只設定 ANTHROPIC_BASE_URL 而無憑證, 訂閱額度仍適用

10. Other LLM gateways 使用既有 LLM gateway

任何公開受支援 API 格式的 gateway 都能搭配 Claude Code, Anthropic 不背書, 維護, 或稽核第三方 gateway 產品, 也不支援路由到非 Claude 模型.

憑證
供應商金鑰留在伺服器端, 開發者持有 gateway 憑證
用量追蹤
依開發者或團隊歸因
成本控管
集中執行預算與速率限制
供應商切換
改 gateway 設定即可, 不需改動開發者機器

gateway 產品需隨 Claude Code 演進持續更新轉發規則, 否則新版本引入的 anthropic-beta 值與請求欄位會被裁切導致功能失效.

12. Gateway protocol reference gateway 協定參考

gateway 必須公開至少一種 API 格式, 且推論回應必須支援串流, 不可整批緩衝後才轉發.

格式對應變數
Anthropic MessagesANTHROPIC_BASE_URL
Bedrock InvokeModelANTHROPIC_BEDROCK_BASE_URL + CLAUDE_CODE_USE_BEDROCK
Agent Platform rawPredictANTHROPIC_VERTEX_BASE_URL + CLAUDE_CODE_USE_VERTEX
關鍵原則

anthropic-version 與 anthropic-beta 標頭必須原封不動轉發, 視為開放清單而非白名單.功能是否正常取決於標頭與請求本文欄位是否成對轉發, 缺一通常產生硬性 400 錯誤.自動重試邏輯依上游錯誤文字比對, gateway 不應把錯誤包進自己的信封格式.

14. Claude apps gateway 總覽

Anthropic 自製, 內建於 claude 執行檔中的自架 gateway, 讓開發者以企業身分識別提供者（IdP）登入, 而非持有 API key 或雲端憑證.以 claude gateway --config gateway.yaml 啟動.

路由
Bedrock, Google Cloud, Foundry, 或 Anthropic API
存取控管
依 IdP 群組強制模型存取與 managed settings
遙測
OTLP 資料送到組織自有可觀測性堆疊
部署需求
v2.1.195 以上, OIDC IdP, Postgres 14 以上, 私有網路位址
限制

不支援伺服器端 web search, 不支援 1 小時快取 TTL, 不支援 SAML/LDAP, 一個 gateway 只服務一個 OIDC 發行者, 不支援 Windows 伺服器.

18. Claude apps gateway spend limits 花費上限

花費上限限制每位開發者透過 gateway 花費的金額, 依日, 週, 月為週期.因 gateway 用單一共用上游憑證轉發所有推論, 若無個別上限, 失控的代理群可能耗盡組織整個額度.

scope.type目標
user依 OIDC sub 指定個別開發者
rbac_group依 IdP 群組, 每位成員各自套用
organization組織預設值

超過上限時下一次請求收到 429, 用量以串流回應中的 token 數乘以 USD 牌價估算, 屬於斷路器而非正式發票.

19. Claude Code on Amazon Bedrock 透過 Bedrock 設定

登入精靈引導完成設定, 執行 claude 於登入畫面選擇 3rd-party platform 再選 Amazon Bedrock.手動設定需先在 Bedrock 主控台送出模型存取申請表, 設定 AWS 憑證, 再設定 CLAUDE_CODE_USE_BEDROCK=1 與 AWS_REGION.

export ANTHROPIC_DEFAULT_OPUS_MODEL='us.anthropic.claude-opus-4-8'
export ANTHROPIC_DEFAULT_SONNET_MODEL='us.anthropic.claude-sonnet-4-6'
export ANTHROPIC_DEFAULT_HAIKU_MODEL='us.anthropic.claude-haiku-4-5-20251001-v1:0'
Mantle

另一種以原生 Anthropic API 格式服務 Claude 模型的 Bedrock 端點, 用 CLAUDE_CODE_USE_MANTLE=1 啟用, Sonnet 5 一律透過 Mantle 以 1M 情境視窗執行.

登入精靈（需 v2.1.98 以上）引導完成驗證與模型確認.手動設定需先啟用 Vertex AI API, 於 Model Garden 申請模型存取（約需 24 至 48 小時核准）, 再設定 CLAUDE_CODE_USE_VERTEX=1, CLOUD_ML_REGION, ANTHROPIC_VERTEX_PROJECT_ID.

21. Claude Code on Microsoft Foundry 透過 Foundry 設定

Foundry 沒有互動式設定精靈, 純以環境變數設定.先在 Foundry 主控台建立 Claude 資源與模型部署, 驗證支援 API key（ANTHROPIC_FOUNDRY_API_KEY）或 Microsoft Entra ID 兩種方式.

export CLAUDE_CODE_USE_FOUNDRY=1
export ANTHROPIC_FOUNDRY_RESOURCE={resource}
重要差異

Foundry 沒有啟動時模型檢查機制, 模型版本未釘選且預設不可用時請求會直接失敗, 而非像 Bedrock, Vertex AI 那樣自動回退舊版本.務必釘選模型並在建立 Azure 部署時選擇特定版本而非自動更新到最新.

22. Claude Code on Claude Platform on AWS 透過 Claude Platform on AWS 設定

Anthropic 直接維運的 Claude API, 搭配 AWS 驗證, IAM 存取控制, AWS Marketplace 計費, 請求直達 Anthropic API, 因此功能與發布節奏與 Claude API 相同.透過 AWS Marketplace 訂閱會建立一個綁定 AWS 帳號的全新 Anthropic 組織.

驗證方式

AWS 憑證搭配 SigV4, 標準 AWS 憑證鏈

Workspace API key, 優先於 SigV4

啟用設定

CLAUDE_CODE_USE_ANTHROPIC_AWS=1

ANTHROPIC_AWS_WORKSPACE_ID, AWS_REGION

Bedrock 與 Foundry 在供應商路由上優先於 Claude Platform on AWS, 同時設定時需取消 CLAUDE_CODE_USE_BEDROCK 與 CLAUDE_CODE_USE_FOUNDRY.

23. Enterprise deployment overview 企業部署總覽與比較

多數組織適合用 Claude for Teams 或 Enterprise, 團隊成員以單一訂閱同時取得 Claude Code 與網頁版 Claude, 集中計費, 不需自建基礎架構.Enterprise 再加上 SSO, 網域認領, 角色型權限, compliance API, managed policy settings.

方案最適合計費
Teams/Enterprise多數組織每席位或聯繫業務
Anthropic Console個別開發者用量計費
Bedrock/Vertex/Foundry雲端原生部署透過該雲端帳單

企業最佳實務包含, 多層級部署 CLAUDE.md 文件, 建立一鍵安裝流程, 鼓勵新使用者先從程式碼問答開始, 為雲端供應商釘選模型版本, 用 managed permissions 設定安全政策, 由中央團隊維護 checked-in 的 .mcp.json.
