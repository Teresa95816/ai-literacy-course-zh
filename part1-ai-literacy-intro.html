AI 入門課

Part 1: 認識你的工作夥伴, 你不需要會寫程式, 也不需要懂技術, 只需要願意開口問.

1-1 神經網路 (Neural Network)

大型語言模型(LLM)是一種利用大量文字資料訓練而成的人工智慧模型. 其底層 Transformer 模型是一組神經網絡, 透過 Transformer 架構與自注意力機制, 學習文字之間的關聯與語意.

比喻

神經網路的命名和設計靈感直接來自人腦.

查看 Neural Network Demo →

1-2 預訓練 (Pre-training)

海量文本壓縮
開發者取得 Internet 上海量的文本, 利用數千個 GPU 持續運行十數日, 最終將這些資訊「壓縮」進一個檔案中.
基礎模型 (Base Model)
= 壓縮檔案 + 一段 C 語言程式碼.

每個模型都有「知識截止日(Knowledge Cutoff)」, 截止日之後的事它一概不知.

1-3 微調 (Fine-tuning) 與對齊 (Alignment)

把基礎模型調教成一個「助手模型」(Assistant Model), 以符合規範的方式和語氣回答問題.

武器不能教人製作武器
仇恨言論不能散布仇恨言論
幻覺不能產生幻覺(Hallucination)而捏造不存在的事實

企業自建 AI 助理最務實的路線, 是檢索增強生成(Retrieval-Augmented Generation, 簡稱 RAG): 把內部文件整理成可以查找的資料庫, 讓 AI 回答前先查, 既比重訓便宜, 資料也隨時可以更新.

1-4 MoE (混合專家模型)

比喻

當你在聽音樂時, 大腦的聽覺皮質最活躍; 當你在解數學題時, 前額葉最忙碌. 人腦不會讓所有神經元同時全力運轉, 它會按需啟動對應的專區.

MoE 模型的設計正是如此: 每次收到輸入, 由「路由器」決定派哪幾個專家子網路處理, 其餘保持靜止, 最後將各專家的輸出加權合併.

知識更廣

各領域各有擅長的專家, 同時涵蓋更多主題.

成本更低

每次只啟動部分資源, 其餘專家保持靜止.

1-5 推論 (Inference)

運行現有模型以生成文字, 每次只預測一個 Token, 預測完再把它加進去, 然後預測下一個, 如此循環直到輸出完整.

Step 1模型預測下一個最可能的 Token
Step 2把這個 Token 加進已生成的序列
Step 3重複以上步驟, 直到輸出完整
概念

Token 是 AI 閱讀和思考的基本單位, 不完全等於「字」或「詞」. OpenAI Tokenizer →

1-6 思維鏈 (Chain of Thought)

2022 年, 研究人員發現只要在 Prompt 中加入「請一步一步思考(Let's think step by step)」等引導語, 模型在數學計算、邏輯推理與常識判斷等任務上的表現便能明顯提升.

選擇時期:
2022: 使用者觸發
2024 起: 內建推理

早期的 CoT 是一種 Prompt Engineering 技巧, 需要由使用者主動觸發, 例如自己在提示詞裡寫上「請一步一步思考」.

許多先進模型已將推理能力內建於模型架構中, 能自動進行多步驟思考, 不需要使用者特別引導.

CoT 產生的推理 Token 會消耗額外運算資源, 因此在部分模型或服務中, 這些 Token 也會納入計費或用量統計.

1-7 無狀態與多輪對話 (Stateless & Multi-turn Conversation)

比喻

電影《我的失憶女友》(50 First Dates)裡, 女主角 Lucy 因車禍每天醒來都會失去前一天的記憶. 男主角 Henry 為了讓她記得兩人的感情, 每天早上準備一捲影帶, 讓她從頭看完所有的故事, 才能繼續當天的對話.

模型一次能「看到」的 Token 數量是有上限的, 稱為上下文視窗(Context Window). 超過這個上限, AI 就會「忘記」對話最前面的內容.

開始一段長工作前, 先在對話開頭貼上「背景摘要」.

1-8 系統提示 (System Prompt)

在你輸入第一個字之前, 產品開發者早已先向 AI 下達一份隱藏指令, 規範它的角色、能力邊界與回應風格. 這份指令稱為 System Prompt(系統提示). Claude API Docs: System Prompts →

System Prompt

產品開發者預先設定, 規範角色與邊界.

就像「員工手冊」.

User Prompt

使用者當下輸入的內容.

就像「當下交辦的工作」.

1-9 工具呼叫 (Tool Use)

早期的大型語言模型只能根據訓練資料生成文字; 而現代 AI 助理則能在需要時主動使用工具, 取得即時資訊、執行計算, 甚至操作外部系統. 以「整理某公司的歷年融資紀錄並製作趨勢圖」為例, AI 會依序完成以下流程:

1 · 網頁搜尋
2 · 計算工具
3 · 資料分析工具
4 · 圖像生成工具

查找公開融資資料.

推算缺失數據或估值.

計算成長率並繪製圖表.

產生相關示意圖片.

步驟 1 / 4

使用者只需描述需求, AI 便能自行判斷何時呼叫工具、如何串接結果, 以及何時將成果回傳給使用者.

1-10 多模態 (Multimodality)

早期語言模型只能處理文字, 現在已能同時處理多種形式的資訊.

看圖 (Vision)
讀取並理解圖片內容, 例如上傳手寫筆記或工作表格照片, AI 直接整理成結構化文件.
生圖 (Image Generation)
根據文字描述產生圖像, 例如輸入產品概念, AI 輸出視覺化設計稿.
語音輸入 (Speech Input)
將口語轉換為文字再處理, 例如會議錄音直接送入 AI 生成摘要.
語音輸出 (Speech Output)
將回答轉換為語音播出, 例如與 AI 進行即時語音對話.
影片理解 (Video Understanding)
分析影片中的畫面與事件, 例如自動整理課程重點、監控異常或比賽內容.
