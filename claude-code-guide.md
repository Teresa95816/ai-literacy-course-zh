# Claude Code

## AI 指令列工具, 給上班族的 Claude Code 入門指南

## 前言

這份文件刻意把範圍限縮在穩定且實用的核心概念上, 不涵蓋進階的技術設定. 原因是 Claude Code 的功能持續更新, 細節異動的速度較快, 此處聚焦在多數人都能立即上手的使用方式, 進階設定請以官方文件為準.

## 什麼是 Claude Code

Claude Code 是 Anthropic 推出的官方指令列(CLI)工具, 讓使用者可以用自然語言下達任務指令, 由 Claude 直接在使用者的電腦上讀取, 建立, 修改檔案, 並執行操作, 完成多步驟任務, 而不只是在對話框中給出文字建議. 除了指令列版本, 目前也提供桌面應用程式(Mac, Windows), 網頁版(claude.ai/code), 以及 IDE 擴充套件(VS Code, JetBrains)等使用方式.

## 與一般聊天機器人的差異

一般的聊天機器人主要在對話框內回答問題或產生文字內容, 使用者仍需要自己動手把結果貼到檔案或系統中. Claude Code 則可以直接操作電腦上的檔案與工具, 例如建立一份 Markdown 文件, 執行版本控制指令, 或搜尋網路資料, 並把結果直接呈現在使用者的工作環境中.

## 安全與權限機制

Claude Code 在執行有風險或難以回復的操作前(例如刪除檔案, 覆寫內容, 對外發送訊息), 會先說明意圖並請求使用者確認, 而不是自行默默執行. 這個機制讓使用者可以隨時掌控 Claude 實際會做的事情, 也是它與一般自動化腳本最大的差異.

## 基本互動方式

使用方式類似於在對話框中打字, 但情境是在終端機(terminal)中進行. 使用者輸入想完成的任務, Claude 會說明接下來打算做的事, 必要時提出確認請求, 完成後回報結果. 任務描述越具體, 得到的結果越準確, 例如指明檔案名稱, 期望的格式或範例, 會比籠統的描述更有效率.

## 內建說明與回饋管道

輸入 `/help` 可以取得使用 Claude Code 的相關說明. 若使用上遇到問題或想提供建議, 可以到以下網址回報:

```
https://github.com/anthropics/claude-code/issues
```

## 可擴充的指令, Skills

Claude Code 支援以 `/` 開頭的指令來呼叫特定功能, 例如協助初始化專案說明文件, 或針對程式碼變更進行審閱. 實際提供哪些指令會依安裝的版本與所安裝的外掛而有所不同, 使用前可以先用 `/help` 確認目前環境支援的項目.

## 職場工作者的應用情境

### 建立與維護文件

可以直接請 Claude Code 協助建立, 修改, 或統一格式化 Markdown 文件, 例如會議記錄, SOP, 教學講義. 本系列前幾份文件(Markdown, Marp, Git & GitHub 教學)就是用這種方式產出的.

### 重複性檔案整理

針對大量檔案的重新命名, 分類, 格式轉換等重複性工作, 可以用自然語言描述需求, 讓 Claude Code 協助執行, 並在每一步給出確認, 減少手動操作出錯的風險.

### 串接版本控制

搭配前面介紹過的 Git, 可以請 Claude Code 協助建立分支, 整理 commit 訊息, 或檢查目前的修改內容, 讓文件與專案的修改歷程更清楚.

### 簡報草稿產出

搭配前面介紹過的 Marp, 可以請 Claude Code 協助撰寫簡報用的 Markdown 草稿, 再自行微調內容與版面.

## 使用時的基本注意事項

1. 涉及刪除, 覆寫, 或對外發送訊息等有風險的操作, 務必先確認內容無誤再核准執行.
2. 處理公司機密或個人資料時, 應遵循所屬企業的資安與保密規範.
3. 大量檔案異動前, 建議先用 Git 做版本控制, 以便需要時可以回復.
4. 任務描述越具體, 結果越精準, 避免使用過於籠統或模糊的指令.
5. 功能與指令會持續更新, 遇到本文件未提及的項目, 以 `/help` 或官方文件為準.
