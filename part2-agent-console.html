Git 和 GitHub 不是同一件事

Git

版本控制工具.

安裝在你的電腦.

在本機記錄每次變更.

GitHub

雲端存放平台.

網站 github.com（微軟旗下）.

把記錄備份到雲端、分享給他人.

關鍵

Git 是工具, GitHub 是平台; GitHub 只是讓 Git 的記錄「上雲」.

核心概念: Repository / Commit / Remote

Repository
簡稱 repo, 是一個有完整歷史記錄的資料夾.
Commit
把當前的變更打包成一個有名字的歷史節點, 附上說明記錄這次改了什麼、為什麼改.
Remote
本機的 repo 只活在你的電腦裡, Remote 是在 GitHub 建立一份備份.
你的電腦(本機 repo)
    ^
    |  pull: 把雲端的最新變更拉回本機
    |  push: 把本機的新 commit 上傳到雲端
    v
遠端 repo(GitHub / GitLab)
    |  pull  ^  push
    v        |
同事的電腦(本機 repo)

GitHub 能帶給你什麼

推上 GitHub, 你得到的遠不只是備份:

雲端備份電腦壞了, 工作還在
跨裝置存取新電腦用 git clone 複製整個 repo, 之後 git pull 拿最新版本
連結即分享把 repo 網址給同事, 他們立刻能看到你的工作
GitHub Pages把 Marp 匯出的 HTML 發布上線, 一個連結就能分享簡報
GitHub Actions固定時間自動執行腳本, 不需要開電腦

Markdown 和 AI 說同一種語言

如果你的工作筆記本來就是 Markdown, 就等於你和 AI 之間不需要格式翻譯. 提示詞、輸出、儲存, 都在同一個格式生態裡流轉.

# 這是標題

這是一段普通文字, *這是斜體*.

- 項目一
- 項目二

這份投影片本身, 就是用 Markdown 寫成的.

# 全域偏好設定

- 在未指定專案目錄的一般對話中（例如當前工作目錄不屬於任何特定專案）, 除非使用者明確要求, 否則只以模型知識為基礎協作, 不主動探索或讀取當前工作目錄以外的任何檔案或專案.
- 不主動觸發任何 custom skill.即使當下對話情境符合某個 skill 的描述, 也不得自行判斷並調用.Skill 的使用必須由使用者明確以 slash command 發起.

## Markdown 生成慣例

- 預設不使用粗體（`**text**`）. 若需強調, 優先改用標題層級、條列結構或程式碼區塊來呈現層次, 而非加粗文字.
- 預設不使用 emoji.除非使用者明確要求, 否則所有回應與檔案內容均不得包含表情符號.
- 預設不使用破折號（— 或 --）.若需連接、轉折或補充說明, 優先改用逗號、句號分句.
- 逗號、句號、分號、冒號統一使用半形（, . ; :）, 其餘標點符號（問號、驚嘆號、括號、引號等）維持全形.
- 半形標點與中文字直接相鄰時, 後方加一個半形空格, 避免視覺擠在一起.

把別人的 repo 複製一份到你自己的帳號.

結果: 你的帳號下多了一份完全獨立的 repo.

Clone

動作發生在你的電腦.

把一份 repo(自己的或別人的)下載到本機.

結果: 本機資料夾裡有一份可以編輯的副本.

開啟 本課程的 GitHub repo, 點擊右上角的 Fork.

Fork 完成後, 網址會變成 github.com/你的帳號/ai-literacy-course-zh, 這是你自己的副本.

點擊綠色的 Code 按鈕, 複製 HTTPS 網址, 在終端機執行:

git clone https://github.com/你的帳號/ai-literacy-course-zh.git
步驟 1 / 3

Clone 下來的 origin, 指向的是你帳號下的副本, 不是原作者的 repo. 之後每一次 commit、push, 都只會進到你自己的 fork, 不會動到原始版本.
