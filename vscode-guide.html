VS Code 入門指南

文字編輯器, 給職場工作者的 VS Code 實用速成課

前言

為什麼職場工作者需要 VS Code

VS Code 是微軟推出的免費文字編輯器, 雖然最初是為程式開發設計, 但因為輕量、跨平台, 且可透過擴充套件擴展功能, 現在也很適合用來撰寫 Markdown 文件、編輯設定檔、管理筆記, 以及搭配 Git 做版本控制.

對於已經接觸過 Markdown、Marp、Git 與 Claude Code 的職場工作者來說, VS Code 正是把這些工具串接在一起的共同工作環境. 學會 VS Code 的基本操作, 就能在同一個介面裡完成文件撰寫、版本控制與 AI 協作等工作.

什麼是 VS Code

Visual Studio Code 是一套免費、開放原始碼、跨平台的文字編輯器. 它本身功能精簡, 但透過安裝擴充套件, 可以擴展出接近完整開發環境的能力.

免費, 開放原始碼
可免費下載與使用, 原始碼公開於 GitHub, 由微軟與社群共同維護
跨平台
支援 Windows、Mac、Linux, 在各作業系統上介面與操作一致
輕量精簡
啟動速度快, 本體功能精簡, 不佔用過多系統資源, 適合日常使用
可擴充
透過安裝擴充套件, 可新增語法高亮、即時預覽、版本控制整合等功能

介面總覽

VS Code 的介面由五個主要區域構成, 各自負責不同的功能. 熟悉這五個區域的位置與用途, 是掌握 VS Code 的第一步.

活動列 (Activity Bar) 畫面最左側的圖示列, 用於切換檔案總管、搜尋、版本控制、擴充套件等主要功能
檔案總管 (Explorer) 顯示目前資料夾中的檔案與子資料夾結構, 點擊即可開啟檔案
編輯區 顯示與編輯目前開啟的檔案內容, 可同時開啟多個分頁或分割成左右兩個視窗
終端機 (Terminal) 內建的指令列視窗, 可在不離開編輯器的情況下執行 Git 指令或其他終端機操作
狀態列 畫面最下方, 顯示目前檔案格式、編碼、游標所在行數位置等資訊

基本操作

開始使用 VS Code 時, 選擇正確的開啟方式至關重要. 建議一律用 "Open Folder" 開啟整個工作資料夾, 而不是只開單一檔案.

Open Folder 開啟整個資料夾 (建議)
  • 可使用檔案總管瀏覽所有檔案
  • 支援整個資料夾全文搜尋
  • 內建 Git 版本控制功能可正常運作
  • 可同時開啟多個分頁與分割視窗
  • 可正確解析專案內的相對路徑
只開單一檔案 (受限)
  • 無法使用檔案總管
  • 搜尋功能限於目前開啟的單一檔案
  • Git 整合功能無法使用
  • 專案間相對路徑連結可能失效
  • 整體功能大幅受限
如何開啟資料夾

選擇選單列的 File → Open Folder, 或在命令面板輸入 "Open Folder", 即可把整個工作資料夾作為 VS Code 的工作區 (Workspace) 開啟.

命令面板 Command Palette

命令面板是 VS Code 最有效率的操作方式之一, 用文字搜尋即可執行幾乎所有功能, 不需要記住每個功能藏在哪一層選單裡.

Ctrl + Shift + P
開啟命令面板
Mac 上以 Cmd 取代 Ctrl
可以在命令面板做什麼
  • 搜尋並執行任何 VS Code 功能或指令
  • 切換顏色主題 (搜尋 "Color Theme")
  • 安裝擴充套件 (搜尋 "Extensions: Install")
  • 開啟設定頁面 (搜尋 "Open Settings")
  • 執行 Git 操作 (搜尋 "Git:")
  • 格式化目前檔案 (搜尋 "Format Document")

擴充套件 Extensions

點擊活動列中的擴充套件圖示, 或在命令面板搜尋 "Extensions: Install Extensions", 即可在線上市集搜尋並安裝套件. 以下四款是與文件工作最直接相關的推薦套件.

內建 Markdown 預覽
不需額外安裝, VS Code 原生支援, 即可即時預覽 Markdown 文件的排版效果
Marp for VS Code
在 VS Code 中撰寫與預覽 Marp 簡報, 並可直接匯出 PDF 或 PPTX 檔案
GitLens
加強內建的 Git 功能, 方便查看每一行程式碼或文字內容的修改紀錄與作者
Claude Code (IDE 擴充套件)
在編輯器內直接與 Claude Code 互動, 不需切換到獨立的終端機視窗
注意

不需要的擴充套件盡量不要安裝. 過多擴充套件會拖慢 VS Code 的啟動與編輯速度. 只安裝目前工作流程中實際會用到的套件.

內建 Git 整合

VS Code 內建支援 Git, 不需要額外安裝套件就能在側邊欄看到檔案變更, 並透過按鈕完成版本控制操作. 下方對比兩種操作方式的等效關係.

VS Code 圖形介面 (點擊操作)
  • 在側邊欄直接看到有變更的檔案清單
  • 點選檔案可比對修改前後的差異
  • 輸入說明文字後點 "Commit" 按鈕
  • 點 "Sync Changes" 完成推送與拉取
等效的終端機指令
git status
git diff 檔案名稱
git commit -m "說明文字"
git push

圖形介面操作的底層概念與 Git 指令列操作完全相同, 只是改用按鈕與滑鼠點擊完成. 對於不熟悉指令列的使用者, VS Code 的 Git 整合是入門版本控制的友善方式.

Markdown 即時預覽

開啟一份 .md 檔案後, VS Code 提供兩種即時預覽方式, 可依個人習慣選擇.

獨立預覽視窗

按下快捷鍵後, 在新的分頁開啟預覽視窗.

Ctrl + Shift + V
開啟獨立預覽分頁
並排同步預覽 (建議)

在旁邊開一個分割視窗同步預覽, 邊寫邊看, 更直覺.

Ctrl + K then V
在右側開啟分割預覽
適用範圍

這兩個快捷鍵適用於一般 Markdown 文件. 若使用 Marp 格式撰寫簡報, 需要改用 Marp for VS Code 擴充套件提供的預覽功能.

主題與外觀

可以在設定中更換顏色主題、圖示主題, 以及調整字型大小與行距. 對於需要長時間閱讀與編輯文件的使用者, 適當調整外觀有助於降低閱讀疲勞.

顏色主題
更換整個介面的配色, 包含亮色與暗色主題, 可從命令面板搜尋 "Color Theme" 切換
圖示主題
更換檔案總管中檔案與資料夾的圖示樣式, 可讓不同類型的檔案更容易辨認
字型大小
在設定中調整編輯區的文字大小 (Editor: Font Size), 提升長時間閱讀的舒適度
行距設定
調整行與行之間的間距 (Editor: Line Height), 影響文字的密度與閱讀節奏

外觀設定只影響 VS Code 本身的顯示方式, 不會改變任何檔案的實際內容. 可以放心嘗試各種主題, 找到最適合自己的配色與字型組合.

職場工作者常見應用情境

以下四種是職場工作者在日常工作中最常用到 VS Code 的場景.

1 · 撰寫文件
2 · 製作簡報
3 · 版本控制
4 · Claude Code

撰寫與維護文件

把會議記錄、SOP、教學講義等 Markdown 文件統一放在一個工作資料夾中, 用 VS Code 開啟整個資料夾管理, 並利用即時預覽確認排版結果.

建議工作流程

建立一個 docs/ 或 notes/ 資料夾 → 用 VS Code 的 "Open Folder" 開啟 → 使用 Ctrl+K V 並排預覽 → 完成後用 Git 提交版本紀錄.

製作與預覽簡報

搭配 Marp for VS Code 擴充套件, 直接在 VS Code 中撰寫簡報用的 Markdown, 即時預覽投影片效果, 完成後再匯出成 PDF 或 PPTX.

所需套件

在擴充套件市集搜尋並安裝 "Marp for VS Code", 安裝後開啟 .md 簡報檔案時, 右上角會出現 Marp 預覽按鈕.

文件版本控制

利用內建的 Git 圖形介面, 在不需要輸入指令的情況下, 完成文件修改的版本紀錄與同步. 與團隊成員協作時更容易追蹤異動.

基本操作步驟

修改文件後 → 在活動列點擊版本控制圖示 → 在輸入框填寫修改說明 → 點 "Commit" → 點 "Sync Changes" 同步到 GitHub.

搭配 Claude Code 處理日常工作

安裝 Claude Code 的 VS Code 擴充套件後, 可以在編輯器內直接請 Claude 協助建立、修改, 或整理文件, 不需要另外切換到獨立的終端機視窗.

在 VS Code 中同時開著文件檔案與 Claude Code 面板, 可以直接指向目前正在編輯的內容, 讓 AI 協作更直覺. 這也是整個工具鏈最終串接起來的樣子.

使用時的基本注意事項

以下五點是新手最容易忽略的地方, 養成習慣後可以避免常見的困擾.

注意事項 1 建議用 "Open Folder" 開啟整個工作資料夾, 只開單一檔案會無法使用檔案總管與全資料夾搜尋等功能
注意事項 2 不需要的擴充套件盡量不要安裝, 過多擴充套件會拖慢啟動與編輯速度
注意事項 3 修改檔案後留意是否已經儲存 — 分頁標題若顯示一個圓點, 代表內容尚未儲存
注意事項 4 內建終端機預設使用的命令列環境 (例如 PowerShell 或 bash) 可能與平常習慣不同, 留意指令語法上的差異
注意事項 5 同時開啟多個視窗或工作區時, 切換前養成隨手存檔的習慣, 避免變更遺失

快速複習表

本指南出現的五個核心名詞與說明.

工作資料夾 (Workspace)

一邊編輯 Markdown 或 Marp 簡報, 一邊同步看到排版結果的功能
內建終端機
不離開編輯器即可使用的指令列視窗, 以 Ctrl+` 開啟或關閉

結語

VS Code 本身學習門檻不高, 只要熟悉開啟工作資料夾、命令面板, 以及搭配 Markdown 預覽與內建 Git 等基本操作, 就能大幅提升日常文件處理的效率.

建議把前面幾份文件 (Markdown、Marp、Git & GitHub、Claude Code) 放進同一個 VS Code 工作資料夾中練習, 體會這些工具如何串接成一套完整的工作流程. VS Code 不只是文字編輯器, 而是讓所有工具協同運作的工作中心.
