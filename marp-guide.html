Marp 入門指南

用文字撰寫簡報, 給職場工作者的 Marp 快速入門

前言

Marp 是一套讓使用者用 Markdown 純文字撰寫簡報的工具生態系. 職場工作者常常需要快速產出會議簡報、教育訓練教材或專案報告, 透過 Marp 可以省去在 PowerPoint 中反覆調整版面的時間, 專注在內容本身, 並能用 Git 等版本控制工具追蹤簡報的修改歷史.

傳統簡報工具的痛點

反覆拖曳文字方塊對齊

版面一改全亂, 花時間重排

版本比對困難, 不易追蹤修改

跨裝置字型與排版不一致

Marp 的解決方式

純文字定義投影片結構與內容

主題樣式與內容完全分離

Git 完整追蹤每次修改歷史

任何設備皆可正確渲染

什麼是 Marp

Marp 全名是 Markdown Presentation Ecosystem, 由 Marp Team 開發與維護. 它讓使用者用一份 Markdown 檔案描述整份簡報, 再透過工具渲染成可投影或可分享的格式. 主要由四個元件組成:

Marp Core
核心渲染引擎, 負責把 Markdown 轉換成簡報用的 HTML 與 CSS
Marp CLI
命令列工具, 可將 Markdown 輸出成 HTML, PDF, PPTX, PNG 等格式
Marp for VS Code
VS Code 擴充套件, 提供即時預覽與匯出功能, 是最常見的入門方式
Marp Web
線上編輯器, 不需安裝即可在瀏覽器中使用, 適合快速嘗試

安裝與啟用方式

Marp 提供三種使用方式, 依照工作習慣選擇最適合的入口:

選擇方式:
VS Code 擴充套件
Marp CLI
Marp Web
步驟 1
在 VS Code 的擴充套件市集搜尋 Marp for VS Code 並安裝
步驟 2
建立一個 .md 檔案, 在最上方加入 marp: true 以啟用渲染模式
步驟 3
點擊編輯器右上角的簡報預覽圖示, 即可即時預覽投影片效果
步驟 4
透過命令面板執行 Marp: Export Slide Deck, 可匯出 PDF, PPTX, PNG, HTML

適合需要自動化批次匯出的場合. 需先安裝 Node.js.

# 全域安裝 Marp CLI
npm install -g @marp-team/marp-cli

# 匯出成 PDF
marp slide.md -o slide.pdf

# 匯出成 HTML
marp slide.md -o slide.html

# 匯出成 PPTX
marp slide.md -o slide.pptx
零安裝

直接在瀏覽器開啟 Marp Web 線上編輯器, 貼上 Markdown 內容即可預覽與匯出. 適合不方便安裝軟體的場合, 例如公司管理的電腦或短暫借用的工作站. 搜尋 "Marp Web" 即可找到官方線上編輯器.

基本架構, 投影片分隔

一份 Marp 簡報由一個 Markdown 檔案組成. 檔案最上方是 YAML 格式的設定區塊(稱為 front matter), 接著用 --- 分隔每一張投影片.

---
marp: true
theme: default
paginate: true
---

# 第一張投影片

這裡是第一張投影片的內容.

---

# 第二張投影片

這裡是第二張投影片的內容.
注意

--- 上下都需要保留空白行, 否則可能無法被正確判斷為投影片分隔線, 而被誤解為其他 Markdown 語法(例如 setext 式標題底線).

全域指令, Front Matter

寫在檔案最上方第一個 --- 區塊內的設定稱為全域指令, 會套用到整份簡報的每一張投影片.

marp: true
啟用 Marp 渲染模式, 為必填項目, 缺少此行工具不會處理這份檔案
theme:
設定主題, 可填入 default, gaia, uncover, 或自訂主題名稱
paginate: true
在每張投影片右下角顯示頁碼
size: 16:9
設定投影片比例, 可填 4:3 或 16:9
backgroundColor:
設定整份簡報的背景色, 填入 CSS 色碼, 例如 #ffffff
color:
設定整份簡報的文字顏色, 填入 CSS 色碼
header:
設定每張投影片共用的頁首文字
footer:
設定每張投影片共用的頁尾文字

區域指令, 單張投影片設定

在某一張投影片內用 HTML 註解格式寫指令, 可以針對單張投影片做客製化設定.

有底線前綴(只套用當張)

指令名稱加上 _ 前綴, 例如 _color, _backgroundColor

效果只套用在這一張投影片, 下一張投影片自動回復原本設定

無底線前綴(套用到後面所有張)

指令名稱不加前綴, 例如 color, backgroundColor

從這張投影片開始, 效果一直延續到整份簡報結束

---

# 一般投影片, 使用預設設定

---

<!-- _backgroundColor: black -->
<!-- _color: white -->

# 這張是黑底白字, 只影響這一張

---

# 這張恢復原本的設定

背景圖片與版面切版

使用 ![bg] 語法可以把圖片設為投影片背景. 點擊下方選項查看各種背景模式的語法與效果:

背景模式:
預設
fit
cover
left
right:40%
![bg](image.jpg)

圖片填滿整張投影片背景, 依比例裁切超出的部分. 等同於 CSS 的 background-size: cover.

![bg fit](image.jpg)

圖片依原始比例完整顯示, 不裁切. 邊緣可能留有空白.

![bg cover](image.jpg)

圖片填滿整張投影片, 維持比例, 超出部分裁切. 與預設行為相同, 可明確標示意圖.

![bg left](image.jpg)

圖片佔左半欄, 右半欄放文字內容. 適合製作圖文對照的說明版面.

![bg right:40%](image.jpg)

圖片佔右側 40%, 百分比可自訂. 文字內容放在左側剩餘空間.

圖片縮放

一般圖片(非背景圖片)可以用 width 與 height 關鍵字調整大小, 不需要另外撰寫 CSS.

# 指定寬度
![width:300px](photo.png)

# 同時指定寬高(縮寫)
![w:300 h:200](photo.png)
width / w
設定圖片寬度, 接受 px 或 auto 等 CSS 單位. 可縮寫成 w.
height / h
設定圖片高度, 接受 px 或 auto 等 CSS 單位. 可縮寫成 h.
同時使用
可在同一組方括號內同時指定 w 與 h, 以空格分隔.

自訂主題

撰寫 CSS 檔案並在開頭加上 /* @theme 主題名稱 */ 即可建立自訂主題, 再透過 front matter 的 theme: 指令套用.

程式碼與數學公式

Marp 支援標準 Markdown 程式碼語法(內建語法高亮), 以及 KaTeX 格式的數學公式.

程式碼

行內程式碼: 用單個反引號包住

區塊程式碼: 用三個反引號包住, 開頭可指定語言名稱觸發語法高亮

數學公式(KaTeX)

行內公式: 用單個 $ 包住

區塊公式: 用兩個 $$ 包住, 獨立一行顯示

# 程式碼區塊
function hello() {
  console.log("Hello, Marp")
}

# 行內數學公式
行內公式: $E = mc^2$

# 區塊數學公式
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$

簡報備忘稿

任何不符合指令格式的 HTML 註解, 會被當作演講者備忘稿(presenter notes). 這些文字只在簡報者模式下才會顯示, 不會出現在正式投影片畫面上.

# 投影片標題

這段是投影片上的正式內容, 觀眾可以看到.

<!-- 這段文字是演講者的提詞, 觀眾不會看到 -->
<!-- 可以記錄補充說明的細節, 或換場白的提示 -->
備忘稿 vs 區域指令

HTML 註解如果符合 Marp 指令格式(例如 <!-- paginate: false -->), 會被當作指令處理. 不符合指令格式的普通文字才會成為演講者備忘稿.

匯出格式

Marp 支援多種匯出格式, 依照使用場合選擇最適合的輸出方式:

HTML
互動式網頁簡報, 可用瀏覽器開啟並透過鍵盤或滑鼠翻頁. 適合線上分享或會議即時呈現
PDF
適合列印, 存檔, 或透過信件寄送. 版面固定, 跨裝置一致
PPTX
匯出後可在 PowerPoint 中繼續微調. 適合需要給同事繼續編輯的場合
PNG / JPEG
每張投影片輸出為獨立圖片, 適合社群媒體或即時通訊分享

職場工作者常見應用情境

三種最常見的辦公場景, 展示 Marp 在實際工作中的使用方式:

1 · 內部會議簡報
2 · 教育訓練教材
3 · 制式報告模板

內部會議簡報

快速產出部門週會或專案進度報告, 頁首直接加入日期與會議名稱, 讓每張投影片都帶有識別資訊.

---
marp: true
theme: default
paginate: true
header: "2026-06-21 部門週會"
---

# 本週重點摘要

- 業績達成率
- 客戶回饋整理
- 下週行動計畫

教育訓練教材

利用 --- 把整套課程拆成多張投影片, 搭配程式碼區塊、表格與圖片, 快速完成講義與簡報合一的教材, 同時用 Git 保留每次修訂的版本紀錄.

版本管理優勢

每次修訂課程內容都能用 git diff 清楚比對改了什麼, 方便多人協作與追蹤修改原因.

制式報告模板

先撰寫一份固定版型的 Markdown 模板, 之後每次只需要替換內容文字, 不需要重新調整版面位置, 可大幅縮短重複性報告的製作時間.

把模板存入版本庫, 團隊成員 clone 後直接填寫內容, 版面樣式由主題統一控管, 不會因為個人習慣而走版.

建議工作流程

從撰寫到發布的四個步驟, 建立可持續維護的 Marp 工作習慣:

1 · 撰寫
2 · 版本控制
3 · 匯出
4 · 呈現

用 VS Code 撰寫 Markdown 內容

安裝 Marp for VS Code 擴充套件, 開啟即時預覽面板, 邊寫邊確認版面效果. 每張投影片的結構在右側立即呈現, 不需要另外開啟程式.

用 Git 管理 .md 檔案

把 .md 檔案納入 Git 版本庫, 每次修改都留下 commit 記錄. 方便追蹤每次內容修改的原因, 也方便多人協作.

匯出成 PDF 或 PPTX

完成後用 Marp CLI 或 VS Code 擴充套件匯出. PDF 適合寄件存檔, PPTX 適合需要在 PowerPoint 中繼續微調的場合.

用 HTML 版本在會議中即時呈現

若需要在會議中即時呈現, 可直接使用匯出的 HTML 版本, 透過瀏覽器全螢幕播放. 鍵盤方向鍵或滑鼠點擊即可翻頁, 不需安裝額外軟體.

常見錯誤提醒

以下是初學者最容易遇到的五個問題:

01
忘記在檔案最上方加入 marp: true, 導致工具沒有啟用 Marp 渲染模式.
02
投影片分隔線 --- 上下沒有保留空白行, 造成分頁失敗或被誤判為其他語法.
03
front matter 的 YAML 縮排寫錯, 導致指令未生效. YAML 對空格非常敏感.
04
區域指令忘記加底線前綴 _, 結果設定被套用到後面所有投影片, 而不是只套用在當前這一張.
05
圖片使用本機絕對路徑, 換到其他電腦或匯出後路徑失效. 建議統一使用相對路徑.

結語

Marp 把寫簡報這件事變成寫一份結構清楚的 Markdown 文件, 對於已經熟悉 Markdown 語法的職場工作者來說, 學習門檻非常低.

第一步
先從簡單的會議簡報開始練習, 熟悉投影片分隔、全域指令與區域指令的用法
第二步
進一步嘗試自訂主題與版面切版, 建立符合公司視覺規範的模板
第三步
逐步把日常的簡報製作流程轉移到 Marp 上, 把省下的時間花在內容品質的提升

Marp 最大的價值不是讓簡報變得更漂亮, 而是讓你可以專注在「說什麼」而不是「怎麼排」. 寫 Markdown, 交給 Marp 處理版面.
