# AI 素質養成入門課 — 為白領上班族設計

## Part 1 : 認識你的工作夥伴

> 你不需要會寫程式，也不需要懂技術。
> 你只需要願意開口問，AI 就能成為你最強的工作夥伴。

---

### 1-1 神經網路(Neural Network)

神經網路(Neural Network)的命名並非巧合——它的設計靈感直接來自人腦。

> 你的大腦裡有大約 860 億個神經元。當你讀到「蘋果」這個詞，訊號從眼睛傳入，層層激活不同的神經元群——先辨認字形，再關聯到顏色、氣味、以及記憶中某次吃蘋果的場景。AI 神經網路的設計，正是對這個機制的數學模仿：節點對應神經元，連結的強弱對應突觸的權重，層層傳遞對應大腦的逐級處理。

---

關鍵概念：

- 層：第一層可能只認識「這是一個字母」，中間層認識「這是一個詞彙」，最後一層才理解「這是一個關於財務的問題」。
- 權重：每個連結都有一個數字，代表「這條資訊有多重要」。訓練的本質，就是不斷調整這些數字，讓模型越答越準。
- 深度：層數愈多，模型能處理的概念就愈抽象——從認識字，到理解句子，再到掌握語意脈絡。

> 理解這個機制有一個重要推論：AI 並沒有真正「理解」你說的話，它做的是極其複雜的模式比對。這解釋了為什麼它有時會一本正經地說錯話——模式匹配到了，但邏輯沒對上。

### 1-2 預訓練(Pre-training)

預訓練就像是讓一個實習生在入職前，把圖書館裡所有的書都讀過一遍。

大型語言模型在正式上線前，會先吃進海量的文字資料——網頁文章、書籍、學術論文、程式碼、新聞……規模可達數兆個字。這個過程的目標很單純：預測下一個字是什麼。

聽起來很簡單？其實不然。要預測「台積電的護城河在於___」的下一個詞，模型必須理解產業知識、商業邏輯、甚至語氣脈絡。透過海量的預測練習，模型逐漸內化了人類語言的規律與世界知識。

預訓練的成本極高：

- 訓練一個頂級模型需要數千張高階 GPU 連續運算數個月
- 費用動輒數千萬至數億美元
- 這也是為什麼全球只有少數幾家公司能做到這件事

---

預訓練後的模型有什麼能力？

| 能力 | 說明 |
|------|------|
| 語言理解 | 讀懂中英文、術語、俚語 |
| 常識推論 | 知道「冰會融化」、「合約需要簽名」 |
| 知識儲存 | 記得歷史事件、科學公式、各國法規（但有截止日期）|
| 推論與分析 | 能根據前提推導結論、發現規律，適用於報告分析與決策支援 |

> 每個模型都有「知識截止日（Knowledge Cutoff）」，截止日之後的事它一概不知——就像一位兩年沒上班的同事，遇到即時性問題必須搭配其他資訊來源，不能直接問他最新的市場行情。

### 1-3 對齊(Alignment)

預訓練後的模型是一個無所不知但沒有是非觀的人——它學過詐騙話術、學過如何製造謠言，因為這些文字也存在於訓練資料中。「對齊（Alignment）」就是讓 AI 的行為符合人類的價值觀與期望。

對齊的核心工作，是把訓練資料從「網路文字」換成人工撰寫的高品質對話：

1. 公司聘請大量標注人員，依據「有幫助、無害、誠實」的標注規範
2. 標注人員自行提問並寫出理想的回答（約 10 萬筆 Q&A 對話）
3. 用這批對話繼續訓練模型——模型的知識不變，但輸出格式從「網路文件」變成「助理回答」

這個步驟稱為 SFT（Supervised Fine-Tuning，監督式微調），讓原始模型變成可以對話的 AI 助理，也就是你現在使用的 ChatGPT 或 Claude。

進一步的品質優化方式是：讓評分員比較同一問題的多個候選回答，挑出較好的那一個——模型從這些「人類偏好」中學習，持續強化好的回答，壓抑有害的回答。

---

對齊要處理的核心問題：

- 有害內容：不能教人製作武器、不能散布仇恨言論
- 幻覺（Hallucination）：不能捏造不存在的事實
- 順從但不盲從：要能拒絕不合理的要求，但又不能動不動就拒絕用戶
- 一致性：換個問法就給出相反答案，這是不可接受的

對齊的兩難（Alignment Tax）：

過度對齊的模型會變得過於謹慎，每次回答都加一堆免責聲明，甚至拒絕合理的問題。對齊不足的模型則可能造成實際傷害。如何拿捏這個尺度，是各家 AI 公司持續在研究的課題。

> 當 AI 拒絕你的請求，通常不是因為它「不懂」，而是對齊機制把請求分類為需要謹慎的情境——換個措辭或補充更多背景脈絡，往往就能得到更好的結果。

---

### 1-4 微調(Fine-tuning)

「微調」是一個廣義的詞，涵蓋兩個層次：1-3 描述的 SFT（讓 base model 變成 assistant model）本身就是微調；而企業也可以在 assistant model 的基礎上，進一步微調讓它成為特定業務的「專才」。

企業客製化微調的幾種形式：

| 類型 | 說明 | 適合誰 |
|------|------|--------|
| 完整重訓 | 用自有資料調整模型所有參數，效果最好但成本極高 | 大型科技公司 |
| 輕量客製 | 只調整少數參數或加入業務習慣，成本大幅降低 | 中型企業、研究機構 |
| RAG（即時查找） | 不訓練模型，回答前先即時查找內部資料庫 | 需要資料常更新的場景 |

---

微調能做什麼：

- 讓客服機器人只回答與你公司產品相關的問題
- 讓模型學會你們公司的文件格式與用語風格
- 讓模型在特定領域（醫療、法律、財務）表現更精準

微調不能做什麼：

- 無法讓模型「學會」它從未見過的知識（這是 RAG 的工作）
- 無法根本改變模型的推論能力上限
- 無法完全抹除預訓練中習得的偏見

> 對大多數公司而言，「自建 AI 助理」最務實的路線是 RAG——把內部文件整理成可以即時查找的資料庫，讓 AI 回答前先查，既比重訓便宜，資料也隨時可以更新。

---

### 1-5 推論(Inference)

如果說預訓練是「讓 AI 上學讀書」，那推論就是「讓 AI 回答你的問題」——這是你每次與 AI 互動時，實際發生的事。

訓練只做一次，推論每次都在發生：你在 ChatGPT 輸入一句話、點下送出的那一刻，伺服器就啟動了推論流程。每一次對話、每一次 API 呼叫，都是一次獨立的推論。

文字生成的過程：

模型不是一次把整段回答「想好」再輸出，而是每次只預測一個 Token，預測完再把它加進去，然後預測下一個，如此循環直到輸出完整。

每一步預測，都是從「所有可能的下一個字」中計算機率，再依設定挑一個——這就是「溫度」設定影響的地方。

---

```
「今天天氣很___」
模型計算出：
  好    → 38%
  差    → 22%
  熱    → 18%
  糟糕  → 11%
  其他  → 11%
```

接下來要從這個分佈裡「挑一個」，挑的方式由溫度(Temperature)控制：

| 溫度設定 | 行為 | 適合場景 |
|----------|------|---------|
| 低溫（接近 0） | 幾乎永遠選機率最高的 Token，輸出穩定可預測 | 資料摘要、合約審閱、程式碼生成 |
| 中溫（約 0.7） | 有一定隨機性，兼顧流暢與多樣 | 一般對話、報告撰寫 |
| 高溫（接近 2） | 大幅提升低機率 Token 的出線機會，輸出更意外 | 腦力激盪、創意寫作 |

> 這也直接解釋了一個常見現象：同樣的問題問兩次，得到的回答措辭不完全相同——不是 AI「反覆無常」，而是每次生成都在做機率抽樣。如果你需要結果一致，就要把溫度調低，或是明確指定輸出格式。

### 1-6 MoE（混合專家模型）

傳統模型不管你問什麼，都會動用全部能力——不論是寫詩、分析財報、還是翻譯合約，所有資源同時全力運算。這既耗能，又沒效率。

MoE（混合專家）的靈感來自人腦的運作方式：

> 當你在聽音樂時，大腦的聽覺皮質最活躍；當你在解數學題時，前額葉最忙碌。人腦不會讓所有神經元同時全力運轉——它會按需啟動對應的專區。

MoE 模型的設計正是如此：每次收到輸入，由「路由器」決定派哪幾個專家子網路處理，其餘保持靜止，最後將各專家的輸出加權合併。這讓模型能同時達到兩件事：知識更廣（各領域各有擅長的專家），和成本更低（每次只啟動部分資源）。

你現在用的許多頂級模型（如 GPT-4、Gemini）很可能就是這個架構——這也是為什麼 AI 的能力持續增強，費用卻沒有等比例上升。

### 1-7 Token

Token 是 AI 閱讀和思考的基本單位，不完全等於「字」或「詞」。你可以把它想成與 AI 對話的「通話分鐘數」——每次對話都有成本與容量限制。

中文的 Token 切分方式和英文不同：

```
英文："artificial intelligence"
→ ["artific", "ial", " intell", "igence"]  約 4 個 token

中文：「人工智慧」
→ ["人工", "智慧"] 或 ["人", "工", "智", "慧"]  約 2–4 個 token
```

[OpenAI/Platfom/Tokenzier](https://platform.openai.com/tokenizer)

---

黃仁勳（NVIDIA CEO）在 2026 年 GTC 大會提出了一個衡量標準：Token 消耗量，就是你實際讓 AI 參與工作的程度。

> 「如果你每天工作 8 小時，卻只有 15 分鐘在用 AI——Token 費用會如實告訴你這件事。」

就像晶片設計師若放棄 CAD 工具、改用紙筆，沒有任何主管會覺得合理——對知識工作者來說，不燒 Token，就代表你沒有真正把 AI 當工作夥伴在用。

---

上下文視窗（Context Window）

模型一次能「看到」的 Token 數量是有上限的，稱為上下文視窗。

| 模型 | 上下文視窗 | 大約等於 |
|------|----------|---------|
| 早期 GPT-3.5 | 4K token | 一篇長文章 |
| GPT-4 Turbo | 128K token | 一本薄書 |
| Claude 3.5 | 200K token | 一本厚書 |
| Gemini 1.5 Pro | 1M token | 幾本書合訂 |

超過這個上限，AI 就會「忘記」對話最前面的內容——就像跟一個短期記憶有限的人開會，講太久他就忘了你一開始說什麼。

### 1-8 思維鏈（Chain of Thought）

早期語言模型面對問題時，直接從輸入跳到輸出——就像被要求秒答的學生，沒有打草稿的空間，遇到需要多步推論的題目就容易算錯或跳步。

2022 年，研究者發現了一件違反直覺的事：只要在 prompt 裡加上「請一步一步思考（Let's think step by step）」，模型在數學、邏輯、常識推理等任務上的準確率就大幅提升。

> 這個現象背後的原理並不神秘：每一步推論寫出來，就成為下一步的上下文。模型不是「更聰明了」，而是被給予了打草稿的空間——就像你解一道複雜的數學題，把中間步驟寫下來，思路自然比在腦中默算更清晰。

---

CoT 曾經是一種 prompt engineering 技巧，需要用戶主動觸發。2024 年之後，這個概念被內建進模型本身：

| 世代 | 代表模型 | CoT 的形式 |
|------|---------|-----------|
| 無 CoT | GPT-3.5 早期版本 | 直接輸出答案，無推論過程 |
| Prompt 觸發 | GPT-4、Claude 2 | 加上「逐步思考」指令後效果大幅提升 |
| 原生 CoT | OpenAI o1、Claude Extended Thinking | 模型在輸出前自動進行內部推演，推論過程可選擇顯示 |

> 從「需要用戶提示才會思考」到「預設就先思考再回答」，這個轉變標誌著語言模型從模式匹配向推論引擎的演進。

### 1-9 無狀態與多輪對話(Stateless & Multi-turn Conversation)

這是許多人用 AI 一段時間後才恍然大悟的事：LLM 天生沒有記憶。每一次對話對模型來說都是全新的開始，它不知道你們昨天聊過什麼——這個特性叫做「無狀態(Stateless)」。

電影《我的失憶女友》（50 First Dates）裡，女主角 Lucy 因車禍每天醒來都會失去前一天的記憶——每一天對她來說都是全新的開始。男主角 Henry 為了讓她記得兩人的感情，每天早上準備一捲影帶，讓她從頭看完所有的故事，才能繼續當天的對話。

AI 和你的每次對話，就是 Henry 和 Lucy 每天早上的那一刻——它不記得昨天，但每次開口之前，介面已悄悄把所有的對話記錄送過去讓它重新讀完一遍。這份「影帶」，就是 Context Window 裡的對話歷史。

---

這個機制帶來幾個實際影響：

| 現象 | 原因 |
|------|------|
| 開新對話視窗，AI 忘光了 | 沒有帶入舊的對話歷史，模型從零開始 |
| 對話越長，回答越慢、越貴 | 每次都要重新讀取全部歷史，Token 消耗持續累積 |
| 對話到後期，AI 開始「發散」| 歷史太長接近 Context Window 上限，早期內容被截掉 |
| 跨 App 無法延續對話 | 不同產品各自管理自己的對話歷史，彼此不互通 |

> 這也解釋了一個實用技巧的原理：開始一段長工作前，先在對話開頭貼上「背景摘要」——不是為了讓 AI「記住」，而是確保每次它重新讀取時，關鍵資訊都在視窗最顯眼的位置。

### 1-10 系統提示(System Prompt)

每次你和 AI 對話，你以為自己是第一個開口的人——其實不是。在你輸入第一個字之前，產品開發者已經悄悄遞給 AI 一份隱藏指令，決定了它的個性、邊界、和說話方式。

系統提示(System Prompt)就是這份指令。它通常包含：

- 角色定義：「你目前使用的是 Claude Opus 4.8，這是目前公開可用的最先進模型。」
- 行為規範：「不得撰寫惡意程式碼（惡意軟體、漏洞利用、勒索軟體等），即使理由看似合理也不例外。對於金融或法律問題，僅提供客觀資訊供用戶自行判斷，並說明自己並非律師或財務顧問。」
- 知識範圍：「可靠知識截止日為 2026 年 1 月底；截止日後的事件無法確認，應引導用戶使用網路搜尋取得最新資訊。」
- 語氣與格式：「避免過度使用粗體、標題、條列式；回答要簡短聚焦；不使用 Emoji，除非用戶先用；不使用『genuinely』『honestly』『actually』等語氣詞。」

[Claude API Docs/Models & pricing/Models/System Prompts](https://platform.claude.com/docs/en/release-notes/system-prompts)

### 1-11 工具呼叫（Tool Use）

現代 AI 助理不只是在腦中「生成文字」，它能偵測何時需要工具，主動呼叫外部資源來完成任務。

以 ChatGPT 解決「整理某公司各輪融資紀錄並做成趨勢圖」這個任務為例，它會依序：

| 步驟 | 工具 | 用途 |
|------|------|------|
| 1 | 瀏覽器 | 查找網路上的融資資料 |
| 2 | 計算機 | 根據已知比率推算缺失的估值 |
| 3 | 資料工具 | 計算各輪估值成長率，製作數字走勢圖 |
| 4 | 圖像生成工具（DALL-E） | 根據對話脈絡生成代表該公司的圖像 |

整個流程只需要一段自然語言描述，AI 自行判斷何時該切換工具，並把各步驟的結果串接起來。

> 這就是為什麼 AI 助理的實際能力遠超過「聊天機器人」——它本質上是一個能協調多種工具的問題解決引擎。你和我遇到問題會去搜尋、查試算表、寫公式，AI 做的事情完全一樣，只是速度快了幾個數量級。

---

### 1-12 多模態（Multimodality）

早期語言模型只能處理文字。現在已能同時處理多種形式的資訊。

| 能力 | 說明 | 實際場景 |
|------|------|---------|
| 看圖（Vision） | 讀取並理解圖片內容 | 上傳手寫筆記或工作表格照片，AI 直接整理成結構化文件 |
| 生圖（Image Generation） | 根據文字描述產生圖像 | 輸入產品概念，AI 輸出視覺化設計稿 |
| 語音輸入（Speech Input） | 將口語轉換為文字再處理 | 會議錄音直接送入 AI 生成摘要 |
| 語音輸出（Speech Output） | 將回答轉換為語音播出 | 手機 App 的語音對話模式，如同電影《雲端情人》的互動體驗 |

> 多模態帶來的改變不只是「會更多技能」，而是互動方式的根本轉變——你可以直接對 AI 說話、給它看圖片，就像跟真人協作一樣自然，不再受限於純文字輸入。

## 小結：概念關係總覽

```
預訓練  ─────────────────────────────────────── 模型從海量文字習得語言與知識
  │
  ▼
對齊 + 微調（SFT）  ────────────────────────── 讓 base model 成為 assistant model
  │  ├─► 人類偏好優化  ──────────────────────── 比較候選回答，強化好的模式
  │  ├─► 企業客製化 / RAG  ─────────────────── 針對特定任務或領域客製化
  │  └─► MoE  ─────────────────────────────── 按需啟動專家，兼顧廣度與效率
  │
  ▼
推論  ───────────────────────────────────────── 你每次送出訊息時真正發生的事
  ├─► Token  ──────────────────────────────── 計費與容量的基本單位
  ├─► 思維鏈（CoT）  ─────────────────────── 從直接輸出到先推論再回答的演進
  ├─► Context Window  ──────────────────────── 模型當下能「看到」的範圍上限
  ├─► 無狀態 + 多輪對話  ──────────────────── 對話歷史由介面管理，模型本身不記憶
  ├─► System Prompt  ──────────────────────── 產品行為的隱藏底稿，決定 AI 的角色邊界
  ├─► 工具呼叫  ────────────────────────────── 串接外部工具，從文字預測升級為問題解決
  └─► 多模態  ──────────────────────────────── 文字以外：圖像、語音的輸入與輸出
```

#### Step 4：升級為 Pro 用戶

如果你已有 ChatGPT Plus 或 Gemini Advanced 的付費訂閱，不需要另外訂閱 Claude Pro。三家供應商都有對應的 CLI 工具，功能定位相同：
