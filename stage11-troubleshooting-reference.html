1. Debug your configuration 除錯設定

當 Claude 忽略指示或設定的功能沒出現, 通常是檔案沒載入, 從非預期位置載入, 或被另一個檔案覆蓋.先執行 /context 檢視情境視窗組成, 確認 CLAUDE.md, rules, skill 描述是否存在.

/memory
已載入的 CLAUDE.md 與 rules 檔案
/hooks
生效中的 hook 設定
/mcp
已連接的 MCP server 與狀態
/doctor
設定診斷, 無效欄位, schema 錯誤

常見陷阱

症狀原因
hook 從不觸發matcher 用了 JSON 陣列而非用 | 分隔的字串
MCP server 不載入設定寫在 .claude/ 底下而非 repo 根目錄的 .mcp.json
子目錄 CLAUDE.md 被忽略需 Claude 讀取該目錄檔案時才載入, 非啟動時載入

用 claude --safe-mode 停用所有客製化做乾淨測試, 若問題消失代表原因在 CLAUDE.md, skills, plugins, hooks, 或 MCP 其中之一.

2. Environment variables 環境變數參考

環境變數控制模型選擇, 驗證, 請求路由, 功能開關, 優先順序為環境變數高於 CLI 旗標/會話指令, 高於設定檔.

選擇分類 :
驗證與端點
模型與超時
功能開關
遙測與除錯

ANTHROPIC_API_KEY, ANTHROPIC_AUTH_TOKEN, ANTHROPIC_BASE_URL, ANTHROPIC_BEDROCK_BASE_URL 等驗證與路由變數.

ANTHROPIC_MODEL, ANTHROPIC_DEFAULT_OPUS_MODEL, API_TIMEOUT_MS（預設 600000ms）, BASH_DEFAULT_TIMEOUT_MS（預設 120000ms）.

CLAUDE_CODE_DISABLE_AUTO_MEMORY, CLAUDE_CODE_DISABLE_FILE_CHECKPOINTING, CLAUDE_CODE_DISABLE_BUNDLED_SKILLS, CLAUDE_CODE_DISABLE_FAST_MODE.

DISABLE_TELEMETRY, DO_NOT_TRACK, CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC（一次停用所有遙測相關項目）, CLAUDE_CODE_DEBUG_LOG_LEVEL.

注意

環境變數在啟動時讀取, 修改後需重啟 Claude Code 才會生效.

3. Error reference 錯誤參考

依錯誤訊息分類查找對應解法, Claude Code 對暫時性失敗會自動以指數退避重試最多 10 次.

分類範例訊息處理方式
伺服器錯誤API Error 500, 529 Overloaded稍後重試, 或用 /model 切換模型
用量限制session limit, Credit balance too low檢查方案額度或帳戶餘額
驗證錯誤Not logged in, Invalid API key/login 重新驗證, 檢查殘留的舊 API key
網路錯誤Unable to connect, SSL 憑證錯誤檢查代理與憑證設定
請求錯誤Prompt is too long, Request too large/compact 或調整請求內容
# 可調整重試行為的環境變數
CLAUDE_CODE_MAX_RETRIES=10      # 重試次數上限
CLAUDE_CODE_RETRY_WATCHDOG=1    # CI 中對 429/529 無限重試
API_TIMEOUT_MS=600000           # 單次請求逾時

5. Troubleshoot installation and login 安裝與登入疑難排解

安裝失敗依症狀對應解法.

1 · PATH 問題
2 · 腳本回傳 HTML
3 · TLS/SSL 錯誤
4 · 登入失敗

command not found 通常是安裝目錄不在 PATH 中（macOS/Linux 為 ~/.local/bin）, 需加入 shell 設定檔.

代表該區域不支援或網路問題, 可改用 Homebrew 或 WinGet 安裝.

需更新系統 CA 憑證或設定 NODE_EXTRA_CA_CERTS 指向企業憑證.

/logout 後重新 /login 可解決多數驗證失敗, 殘留的舊 ANTHROPIC_API_KEY 需 unset.

雲端供應商

Bedrock, Vertex, Foundry 憑證載入失敗通常是雲端 CLI 尚未在目前 shell 完成驗證（aws sts get-caller-identity, gcloud auth application-default login, az login）.

6. Troubleshooting 效能與穩定性疑難排解

處理效能, 穩定性, 搜尋相關問題.

高 CPU/記憶體用 /compact 縮小情境, 用 --safe-mode 排除外部因素, /heapdump 產出快照
自動壓縮 thrashing某檔案反覆撐爆情境, 分段讀取或移到獨立 subagent
指令卡住Ctrl+C 取消, 或關閉後用 --resume 接續 session
終端機亂碼GPU 渲染問題, /terminal-setup 關閉 GPU 加速
搜尋異常ripgrep 無法執行, 安裝系統套件並設 USE_BUILTIN_RIPGREP=0

遇到未列出的問題, 執行 /doctor 做一次性完整診斷, 或用 /feedback 直接回報.
