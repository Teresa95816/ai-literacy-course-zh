1. Checkpointing

Claude Code 會自動追蹤 Claude 所做的檔案編輯, 讓你可以快速復原變更並回到先前的狀態.每個使用者提示詞都會建立一個新的 checkpoint, 並在 30 天後隨 session 自動清除.

rewind 選單的六種動作

選擇動作 :
還原程式碼與對話
還原對話
還原程式碼
從此處摘要
摘要至此處

把程式碼與對話都回復到選定的時間點, 是最完整的還原方式.

回到該訊息但保留目前的程式碼狀態, 適合只想撤回對話方向但不想丟失已完成工作的情境.

復原檔案變更但保留對話內容, 適合程式碼走偏但討論脈絡仍值得保留.

把這個時間點之後的對話壓縮成摘要, 釋放情境視窗空間, 適合丟棄離題討論但保留早期完整細節.

把這個時間點之前的對話壓縮成摘要, 保留之後的訊息不動, 適合壓縮早期設定討論同時保留近期完整工作內容.

# 開啟 rewind 選單
/rewind
# 或在提示輸入框空白時連按兩次 Esc
限制

checkpointing 不會追蹤由 bash 指令（如 rm、mv、cp）修改的檔案, 也不會追蹤 Claude Code 之外手動修改的檔案.

checkpoint 是為快速的 session 層級復原而設計, 不能取代版本控制, 可視為 "本機復原", Git 才是 "永久歷史".

2. Code Review

Code Review 分析 GitHub pull request, 由一組專門的代理在完整程式碼庫情境中檢視變更, 並以行內留言回報邏輯錯誤、安全漏洞、邊界情況錯誤與細微迴歸問題.目前為研究預覽階段, 適用於 Team 與 Enterprise 訂閱方案.

嚴重程度標記

選擇等級 :
Important
Nit
Pre-existing

紅色標記, 合併前應修復的錯誤.

黃色標記, 值得修但不阻擋合併的小問題.

紫色標記, 程式碼庫中既有但非此 PR 引入的錯誤.

設定 Code Review

2 · 安裝 GitHub App

4 · 設定觸發時機

需要 Owner 或 Primary Owner 角色, 在 claude.ai/admin-settings/claude-code 找到 Code Review 區塊.

安裝 Claude GitHub App, 需要 Contents 讀寫、Issues 讀寫、Pull requests 讀寫權限.

選擇要啟用 Code Review 的儲存庫, 之後可再新增.

每個儲存庫可選擇 PR 建立後一次、每次推送後、或僅手動觸發（留言 @claude review）.

4. 安全性

Claude Code 建置時將安全性視為核心, 預設採用嚴格的唯讀權限, 需要額外動作時會明確請求權限.

內建保護機制 與 提示注入防護

內建保護機制

沙盒化的 bash 工具, 隔離檔案系統與網路存取

寫入權限限制在啟動時所在的資料夾及子資料夾

提示疲勞緩解, 支援允許清單設定

Accept Edits 模式自動核准常見檔案系統指令

提示注入防護

權限系統, 敏感操作需明確核准

情境感知分析偵測潛在有害指示

網路指令核准, curl、wget 不自動核准

隔離的情境視窗, WebFetch 使用獨立情境

雲端執行安全性

隔離的虛擬機器每個雲端 session 在獨立的 Anthropic 管理 VM 中執行
憑證保護驗證透過安全代理處理, 使用範圍受限的沙盒憑證
分支限制git push 操作限於目前工作分支
稽核日誌雲端環境中所有操作都會被記錄
最佳實務

核准前檢視所有建議變更, 對敏感儲存庫使用專案專屬權限設定, 考慮使用 dev containers 做額外隔離, 定期用 /permissions 稽核權限設定.

5. 在 Claude 撰寫程式碼時抓出安全問題 security-guidance

security-guidance 外掛讓 Claude 在工作過程中審視自己所做的程式碼變更是否有常見漏洞, 並在同一個 session 中修復, 安裝後自動運作, 不需另外呼叫指令.

三層檢查

1 · 每次檔案編輯
2 · 每回合結束
3 · 每次 commit / push

快速的模式比對, 尋找危險呼叫（如 eval、pickle、dangerouslySetInnerHTML）, 不呼叫模型, 不計成本.

背景執行的模型審核, 檢視該回合改動的一切, 抓出授權繞過、注入、伺服器端請求偽造、弱加密等問題.

更深入的代理式審核, 會閱讀周邊程式碼（呼叫端、消毒函式、相關檔案）以判斷發現是否真實, 每小時上限 20 次.

自訂規則範例

# .claude/security-patterns.yaml
patterns:
  - rule_name: internal_api_key
    substrings: ["sk_live_", "AKIA"]
    reminder: "Hardcoded API key prefix."
停用方式

可個別停用各層, ENABLE_PATTERN_RULES=0、ENABLE_STOP_REVIEW=0、ENABLE_COMMIT_REVIEW=0, 或以 SECURITY_GUIDANCE_DISABLE=1 完全停用.

6. 資料使用

Anthropic 依帳號類型與偏好保留 Claude Code 資料.

資料訓練政策

選擇使用者類型 :
消費者使用者
商業使用者

Free、Pro、Max 方案可選擇是否允許資料用於改進未來的 Claude 模型, 允許者資料保留 5 年, 不允許者保留 30 天.

Team、Enterprise 方案、API、第三方平台維持既有政策, Anthropic 不會用商業條款下傳送的程式碼或提示詞訓練模型, 除非客戶已選擇加入 Development Partner Program, 標準保留期為 30 天.

擷取 URL 前只會把主機名稱送到 api.anthropic.com 核對安全封鎖清單, 不含完整 URL、路徑或頁面內容, 結果依主機名稱快取 5 分鐘.

7. 法律與合規

使用 Claude Code 受商業條款（Team、Enterprise、Claude API 使用者）或消費者服務條款（Free、Pro、Max 使用者）規範.

授權條款依帳號類型適用商業條款或消費者服務條款
醫療合規 BAA客戶簽署 BAA 且啟用 Zero Data Retention 後自動延伸涵蓋 Claude Code
OAuth 驗證僅供訂閱方案購買者做一般使用, 不可代表使用者路由請求
API 金鑰驗證開發者建置互動產品應改用 Console 或雲端供應商的 API 金鑰
安全漏洞回報透過 HackerOne 管理的安全計畫回報

無論是直接使用 Claude API（1P）或透過 Amazon Bedrock、Google Vertex 存取（3P）, 既有的商業協議都適用於 Claude Code 的使用, 除非雙方另有協議.
