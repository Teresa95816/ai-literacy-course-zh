4. Desktop application

充分運用 Claude Code Desktop, 平行 session, computer use, Dispatch 與企業設定

Desktop app 有三個分頁 : Chat, Cowork（Dispatch 與長時間代理工作）, 以及 Code（軟體開發, 本節重點）.

選擇權限模式

點選查看說明 : Ask permissions Auto accept edits Plan mode Auto Bypass permissions
行為Claude 編輯檔案或執行指令前會詢問, 適合新手
行為自動核准檔案編輯與常見檔案系統指令（mkdir, touch, mv）, 其他終端機指令仍會詢問
行為Claude 讀取檔案並執行指令探索, 提出計畫但不編輯原始碼
行為執行所有動作並附帶背景安全檢查, 減少權限提示同時維持監督
行為不出現權限提示（除了明確的 ask 規則）, 僅限沙箱化容器或 VM 使用

雲端 session 僅支援 Accept edits, Plan mode 與 Auto 模式.

電腦操作（Computer use）權限層級

僅檢視
只能在截圖中看到 app, 適用於瀏覽器, 交易平台
僅點擊
可點擊與捲動, 但不能打字, 適用於終端機, IDE
完全控制
可點擊, 打字, 拖曳, 使用快速鍵, 適用於其他所有 app

CLI 與 Desktop 對照

CLI

支援所有權限模式（含 dontAsk）

用 --worktree 手動隔離 session

支援腳本化與 Agent SDK

支援 agent teams

Desktop

Ask, Auto accept, Plan, Auto, Bypass（設定中啟用）

自動建立 worktree

不支援腳本化 / --print

不支援 agent teams, 改用 dynamic workflows

執行 /desktop 可把終端機的 CLI session 移到 Desktop app 中繼續.

6. Use Claude Code with Chrome (beta)

連接 Claude Code 到 Chrome 瀏覽器, 測試網頁應用, 除錯, 自動化表單填寫並擷取資料

Claude 會為瀏覽器任務開啟新分頁, 並共享你的瀏覽器登入狀態. 瀏覽器動作即時在可見的 Chrome 視窗中執行; 遇到登入頁或 CAPTCHA 時會暫停並請你手動處理. 目前僅支援 Google Chrome 與 Microsoft Edge.

claude --chrome
# 或在既有 session 內執行 /chrome

範例工作流程

選擇範例 : 測試本機網頁應用 用 console log 除錯 自動化表單填寫 擷取網頁資料

"I just updated the login form validation. Can you open localhost:3000, try submitting the form with invalid data, and check if the error messages appear correctly?"

"Open the dashboard page and check the console for any errors when the page loads." 告知 Claude 要尋找的特定樣式, 而非要求全部 console 輸出.

"I have a spreadsheet of customer contacts in contacts.csv. For each row, go to the CRM at crm.example.com, click 'Add Contact', and fill in the name, email, and phone fields."

"Go to the product listings page and extract the name, price, and availability for each item. Save the results as a CSV file."

常見錯誤 : "Browser extension is not connected"（重啟 Chrome 與 Claude Code 後執行 /chrome）, "No tab available"（請 Claude 建立新分頁）, "Receiving end does not exist"（extension service worker 閒置, 執行 /chrome 重新連線）.

開啟 MCP 選單

11. Voice dictation

在 Claude Code CLI 中用按住錄音或點按錄音的方式口述提示

你的語音會即時轉錄進提示輸入框, 可與打字混用. 需要 Claude.ai 帳號, 本機麥克風, 且組織未啟用 HIPAA 合規設定.

Hold mode（按住錄音, 預設）

按住 Space 開始錄音, 放開後結束並定案

有短暫暖機時間偵測按鍵重複

設定 autoSubmit: true 放開後直接送出

Tap mode（點按錄音與送出）

輸入框為空時點一下開始, 再點一下停止

無暖機時間

轉錄字數達 3 個字以上自動送出

/voice        # 切換開關, 保留目前模式
/voice hold   # 啟用 hold 模式
/voice tap    # 啟用 tap 模式
/voice off    # 停用

語言設定與 Claude 回應語言共用 language 設定, 支援 20 種語言. 按鍵綁定為 voice:pushToTalk, 預設 Space, 可於 ~/.claude/keybindings.json 重新綁定.

12. Launch sessions from links

從 URL 開啟 Claude Code 終端機 session

深層連結是 claude-cli:// 格式的 URL, 會在新終端機視窗中開啟 Claude Code, 並可帶入工作目錄與預先填入的提示. 提示只會填入, 不會自動送出.

建立連結的參數

q
要預先填入提示框的文字, 需 URL 編碼, 最多 5000 字元
cwd
要作為工作目錄的絕對路徑
repo
GitHub owner/name slug, 解析到 Claude Code 曾使用過的本機複本
# cwd 與 repo 都提供時, cwd 優先
claude-cli://open?repo=acme/payments&q=Investigate%20the%20failed%20deploy

用 cwd 適合大家都在相同絕對路徑的情況（如標準化的 devcontainer）; 用 repo 適合每個人複製到不同位置的共用連結. Claude Code 首次啟動互動式 session 時會自動向作業系統註冊 claude-cli:// 處理常式.

13. Share session output as artifacts

把 Claude Code 的工作成果變成可分享的即時互動頁面

Artifact 是 Claude Code 從 session 發布到 claude.ai 私有 URL 的即時互動網頁, 隨著 session 進行即時更新. 需要 Team 或 Enterprise 方案, 並以 /login 登入.

你可以建立什麼

選擇範例 : 走查一項變更 比較替代方案 用互動控制項調整 追蹤進行中的工作

"Make an artifact that walks through this PR. Render the diff with margin annotations and color-code findings by severity."

"Make an artifact with four distinctly different layouts for the settings panel. Vary density and grouping."

"Build an artifact with sliders for the easing curve, duration, and delay so I can try values on this transition."

"Turn this migration plan into a checklist artifact. Check items off as you complete them."

頁面限制

限制影響
不能對外請求CSP 封鎖外部腳本, 樣式, 字型, 圖片, fetch, XHR, WebSocket
沒有後端無法儲存表單資料, 驗證檢視者, 或在檢視時呼叫 API
單一頁面相對連結不會解析, 改用頁內錨點
來源檔案類型必須是 .html, .htm 或 .md
渲染後大小必須小於等於 16 MiB
重點

Artifact 是工作成果的快照, 不是應用程式, 沒有後端且只有你可以編輯; 分享範圍僅限於你的組織內部.
