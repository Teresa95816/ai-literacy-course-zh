Markdown 入門指南

文件排版標記語言, 給職場工作者的實用速成課

前言

什麼是 Markdown

Markdown 是一種輕量級的標記語言, 用最少的符號就能完成排版, 同時保留純文字的可讀性. 對於需要撰寫會議記錄、專案文件、技術筆記或知識庫的職場工作者來說, 學會 Markdown 可以大幅提升文件產出的效率與一致性.

本指南從基礎語法開始, 逐步介紹 Markdown 常用的標記方式, 並提供辦公場景中的實際應用範例. 學完之後, 你就能用純文字符號快速寫出結構清晰的文件.

什麼是 Markdown

Markdown 由 John Gruber 於 2004 年提出, 目的是讓人可以用簡單易讀的純文字格式寫作, 再透過轉換工具產生 HTML 或其他格式的文件. 點選下方類別, 了解各場景的使用情況.

使用場景: 程式碼平台 筆記工具 通訊軟體 靜態網站

程式碼平台 — GitHub / GitLab

這是 Markdown 最普及的使用場景, 幾乎所有開放原始碼專案都用 Markdown 撰寫文件.

  • README.md — 專案首頁說明文件
  • Issue / Pull Request 描述與留言
  • Wiki 頁面與專案文件站

筆記工具 — Notion / HackMD / Obsidian

這類工具支援即時預覽, 讓你一邊輸入 Markdown 一邊看到格式化結果, 適合個人筆記與團隊知識庫.

  • Notion — 支援 Markdown 快捷輸入, 搭配資料庫功能
  • HackMD — 即時多人協作共筆, 適合會議現場
  • Obsidian — 雙向連結筆記, 適合個人知識管理

通訊軟體 — Slack / Microsoft Teams

訊息中使用 Markdown 格式可以讓重點更清楚, 也更容易閱讀長篇說明.

  • *文字* 在 Slack 中顯示為斜體
  • **文字** 在 Teams 中顯示為粗體
  • 反引號包住文字可呈現行內程式碼樣式

靜態網站產生器 — Hugo / Jekyll / Hexo

這類工具讀取 Markdown 檔案並自動轉換為 HTML 頁面, 讓你只需專注寫作, 不需手動維護 HTML.

  • Hugo — 速度極快, 適合大型文件站
  • Jekyll — GitHub Pages 原生支援, 適合個人部落格
  • Hexo — 以 Node.js 為基礎, 臺灣社群廣泛使用

段落與換行

段落之間只要保留一行空白即可分段. 若要在同一段落內強制換行, 有兩種方式可以選擇.

方法一 — 行尾兩個空白

在行尾加上兩個空白字元後按 Enter, 可以在同段落內換行. 純文字編輯器中最常用的方式.

這是第一行  
這是第二行
方法二 — <br> 標籤

直接插入 HTML 的 <br> 標籤, 效果與行尾兩個空白相同. 語意更明確, 但可讀性稍差.

這是第一行<br>
這是第二行
注意

直接按一次 Enter 換行在大多數 Markdown 渲染器中不會產生換行效果, 仍會連接在同一段落. 要真正分段, 需要空一整行.

強調語法

Markdown 提供四種強調方式, 分別對應斜體、粗體、粗斜體與刪除線.

*文字* 或 _文字_
斜體 — 文字會以斜體顯示, 適合強調術語或引用
示範文字以斜體顯示
**文字** 或 __文字__
粗體 — 文字會以粗體顯示, 適合強調重點
示範文字以粗體顯示
***文字***
粗斜體 — 文字同時呈現粗體與斜體, 用於特別強調
示範文字粗體加斜體
~~文字~~
刪除線 — 文字中間加上一條橫線, 常用於標示已刪除或過時的內容
示範文字加上刪除線

清單

Markdown 支援無序清單與有序清單, 兩者都可以加入子項目, 讓結構更清晰.

項目清單 (無序)
編號清單 (有序)

使用 -、* 或 + 開頭, 後面加一個空白. 子項目用兩個空白縮排.

- 第一項
- 第二項
  - 子項目一
  - 子項目二
- 第三項

使用數字加句點開頭. Markdown 渲染器會自動排序, 即使全部寫 1. 也會渲染為 1, 2, 3.

1. 第一步
2. 第二步
3. 第三步
小技巧

有序清單的數字大多會自動排序, 所以可以全部寫成 1., 渲染後仍會顯示正確的 1, 2, 3. 這樣在插入新項目時不需要重新編號.

連結與圖片

連結與圖片的語法相似, 差別只在圖片前面多一個驚嘆號 !.

超連結

方括號放顯示文字, 圓括號放網址. 引號內可加提示文字 (滑鼠停留時顯示).

[顯示文字](網址 "提示文字")

範例:
[Anthropic 官網](https://www.anthropic.com)
圖片

語法與連結相同, 前面加 !. 方括號內是圖片的替代文字 (alt text), 圖片無法顯示時會出現.

![替代文字](圖片網址 "提示文字")

範例:
![公司 Logo](./images/logo.png)
注意

圖片或連結若使用本機路徑 (例如 C:\Users\...), 換到其他環境後路徑會失效. 建議使用相對路徑或完整的網路網址.

引言區塊 Blockquote

使用 > 開頭可以建立引言區塊, 也可以多層巢狀, 常用於引述來源或標示重要備注.

> 這是引言.
>> 這是引言中的引言.

渲染後的效果如下:

這是引言.
這是引言中的引言.

引言區塊在辦公場景中很適合用來引述客戶回饋、會議決議, 或標示需要特別注意的事項, 讓讀者一眼就能區分引述內容與正文.

表格

使用豎線 | 分隔欄位, 第二列是分隔列, 決定欄位的對齊方式.

| 欄位一 | 欄位二 | 欄位三 |
| --- | :---: | ---: |
| 靠左 | 置中 | 靠右 |
欄位一 (靠左) 欄位二 (置中) 欄位三 (靠右)
內容 A 內容 B 內容 C
內容 D 內容 E 內容 F
對齊語法

--- 預設靠左 / :---: 置中 / ---: 靠右. 分隔列的 --- 數量必須與欄位數量一致, 否則表格無法正確渲染.

待辦清單 Task List

許多平台 (GitHub, Notion, HackMD) 支援待辦清單語法. 試著點擊下方的項目, 切換完成狀態.

- [ ] 尚未完成的任務
- [x] 已經完成的任務

互動示範 — 點擊項目可切換狀態:

完成本週會議記錄
更新專案 README 文件
建立團隊知識庫資料夾結構
完成 Markdown 語法學習

跳脫字元

若想顯示 Markdown 語法符號本身而不被解讀為格式, 在符號前加上反斜線 \.

\*這段文字不會變成斜體\*
\# 這不會被渲染成標題
\[這不是連結\]
可跳脫的符號

常見可跳脫的符號: \ ` * _ {} [] () # + - . !. 在這些符號前加上 \ 即可讓它們以原始字元顯示.

職場工作者常見應用情境

以下三種是職場工作者最常用到 Markdown 的場景, 搭配完整範例說明.

會議記錄
專案 README
知識庫筆記

用標題區分出席人員、討論事項, 用待辦清單追蹤行動項目, 讓會議記錄結構清晰、責任明確.

# 2026-06-21 專案進度會議

## 出席人員
- 王小明 (主持人)
- 林小美

## 討論事項
1. 上週進度回顧
2. 本週待辦事項
3. 風險與待解決問題

## 行動項目
- [ ] 王小明: 完成需求文件
- [ ] 林小美: 確認供應商報價

專案說明文件 (README) 是別人對你專案的第一印象. 用 Markdown 可以快速建立標準化的文件結構.

# 專案名稱

## 簡介
這個專案的目的是...

## 安裝方式
`npm install`

## 使用方式
說明如何執行與設定.

## 貢獻方式
1. Fork 此專案
2. 建立功能分支
3. 送出 Pull Request

利用標題分層、清單與表格, 可以快速整理 SOP、操作手冊或常見問題集, 方便團隊成員查閱與維護.

長期將日常工作轉換為 Markdown 格式記錄, 搭配 HackMD 或 Notion 等工具, 可以累積出可搜尋、可維護的團隊知識資產.

常見錯誤提醒

以下是初學者最常遇到的問題, 在寫文件前先牢記這幾個陷阱, 可以省下不少除錯時間.

錯誤 1 標題符號 # 後面忘記加空白, 例如 #標題 而非 # 標題, 導致無法正確渲染
錯誤 2 清單項目與上一行之間沒有空白行, 造成清單無法正確顯示, 與上方段落連在一起
錯誤 3 表格分隔列 --- 數量與欄位數量不一致, 導致整個表格排版錯誤
錯誤 4 中英文字混排時忘記加空白, 在部分平台會影響排版美觀, 例如 使用Markdown寫作 應改為 使用 Markdown 寫作
錯誤 5 圖片或連結使用本機絕對路徑 (如 C:\Users\...), 換到其他環境或上傳至網路後路徑會失效

結語

Markdown 的語法不多, 但只要熟練常用的標題、清單、連結、程式碼與表格語法, 就足以應付絕大多數的辦公文件需求.

建議在日常工作中多加練習, 逐步將會議記錄、專案文件與知識庫筆記都轉換成 Markdown 格式. 長期下來, 文件管理的效率會大幅提升, 而且格式一致的文件也更容易在團隊間流通與維護.
