1. Agent SDK 總覽

Agent SDK 讓你以 Claude Code 作為函式庫建置正式環境的 AI 代理, 提供與 Claude Code 相同的工具、代理迴圈、情境管理能力, 可用 Python 與 TypeScript 程式化操作.內建工具包含 Read、Write、Edit、Bash、Monitor、Glob、Grep、WebSearch、WebFetch、AskUserQuestion, 讓代理不需要你自己實作工具執行邏輯即可開始工作.

安裝

# TypeScript
npm install @anthropic-ai/claude-agent-sdk

# Python, 需 Python 3.10 以上
pip install claude-agent-sdk

與其他介面的差異

介面特性
Client SDK直接的 API 存取, 你自己實作工具執行迴圈
Agent SDKClaude 自主處理工具, 與 Claude Code 同樣的代理迴圈
Claude Code CLI能力相同但為互動式介面, 適合開發與一次性任務
Managed Agents託管的 REST API, 由 Anthropic 代管沙盒與基礎架構
核心能力

內建工具, hooks（生命週期關鍵點執行自訂程式碼）, subagents（專門代理處理聚焦子任務）, MCP（連接外部系統）, permissions（控制工具使用範圍）, sessions（跨多輪維持情境, 可回復或分支）.也支援 Amazon Bedrock、Claude Platform on AWS、Google Vertex AI、Microsoft Azure 等第三方供應商.

2. Quickstart 快速入門

這份快速入門引導建置一個能找出並修復程式錯誤的代理, 前置需求為 Node.js 18 以上或 Python 3.10 以上, 以及一個 Anthropic 帳號.

    options=ClaudeAgentOptions(

        permission_mode="acceptEdits",
    ),
):
    ...

3. Agent loop 代理迴圈如何運作

Agent SDK 讓你在自己的應用程式中嵌入 Claude Code 的自主代理迴圈, Claude 評估提示詞, 呼叫工具採取行動, 接收結果, 重複直到任務完成.

1 · 接收提示詞
2 · 評估並回應
3 · 執行工具
4 · 重複
5 · 回傳結果

SDK 產出 SystemMessage, subtype 為 init.

Claude 可能回應文字、要求工具呼叫、或兩者皆有.

SDK 執行每個要求的工具並收集結果.

步驟持續直到 Claude 產出沒有工具呼叫的回應.

產出最終的 AssistantMessage 加上含最終文字、token 用量、成本、session ID 的 ResultMessage.

回合與情境

一個回合（turn）是迴圈內一次來回, 可用 max_turns 限制工具使用回合數, max_budget_usd 依支出上限停止.effort 選項控制 Claude 投入多少推理, low 用於檔案查詢, medium 用於例行編輯, high 用於重構除錯, xhigh 與 max 用於需要深度分析的多步驟問題.情境視窗不會在回合之間重置, 系統提示詞、CLAUDE.md、工具定義、對話歷史都會累積, 接近上限時 SDK 會自動壓縮較舊的歷史.

4. 在 SDK 中使用 Claude Code 功能

Agent SDK 建構於與 Claude Code 相同的基礎上, SDK 代理可存取相同的檔案系統型功能, 專案指示（CLAUDE.md 與規則）、skills、hooks 等.

settingSources 三種來源

選擇來源 :
project
user
local

載入專案 CLAUDE.md、.claude/rules/*.md、專案 skills、專案 hooks、專案 settings.json.

載入使用者層級 CLAUDE.md 與 skills（~/.claude/）.

載入 CLAUDE.local.md 與本機 settings.json.

省略 settingSources 等同於 ["user", "project", "local"].管理政策設定、~/.claude.json 全域設定、auto memory、claude.ai MCP connectors 不受此控制, 除非個別停用.多租戶隔離務必設定 settingSources: [] 加上 CLAUDE_CODE_DISABLE_AUTO_MEMORY=1.

6. Stream responses 即時串流回應

預設情況下, Agent SDK 在 Claude 完成每個回應後才產出完整的 AssistantMessage.要接收逐步增量更新, 需設定 include_partial_messages（Python）或 includePartialMessages（TypeScript）為 true.

message_start新訊息開始
content_block_start新內容區塊開始, 文字或工具使用
content_block_delta內容的增量更新, text_delta 用於顯示文字生成過程
content_block_stop內容區塊結束
message_delta / message_stop訊息層級的更新與結束

工具呼叫也會逐步串流, 可追蹤工具何時開始、接收其輸入、以及何時完成.

7. Work with sessions 使用 session

session 是 SDK 在代理工作時累積的對話歷史, 包含提示詞、每次工具呼叫、每個工具結果、每個回應, SDK 會自動寫入磁碟供之後回到該 session.

操作行為
continue找到目前目錄中最近的 session 並接續加入新內容
resume需要具體的 session ID 接續既有 session
fork建立以原始歷史複本開始的新 session, 原始 session 保持不變

Python 的 ClaudeSDKClient 會在呼叫之間內部處理 session ID, TypeScript 則傳入 continue: true 讓 SDK 自動挑選目前目錄中最近的 session.跨主機回復 session 需要移動 session 檔案或搭配 SessionStore 轉接器使用共用儲存.

8. Persist sessions to external storage 持久化 session 至外部儲存

預設情況下, SDK 把 session 逐字稿寫入本機檔案系統的 ~/.claude/projects/ 下的 JSONL 檔案.SessionStore 轉接器讓你把這些逐字稿鏡射到自己的後端, 例如 S3、Redis、或資料庫.

append（必要）
寫入新的逐字稿內容
load（必要）
讀回既有的逐字稿
listSessions（選用）
列出可用的 session
delete / listSubkeys（選用）
刪除或列出子鍵
重要

儲存是鏡射而非取代, Claude Code 子行程一律先寫入本機磁碟, 鏡射寫入是盡力而為的, 若失敗會重試至多三次, 本機逐字稿依然完整不受影響.SDK 提供 InMemorySessionStore 供開發測試, 也在 TypeScript SDK 儲存庫提供 S3、Redis、Postgres 的參考轉接器.

9. Give Claude custom tools 自訂工具

自訂工具讓你擴充 Agent SDK, 定義自己的函式供 Claude 呼叫, 透過 SDK 的行程內 MCP server 存取資料庫、外部 API、或領域專屬邏輯.

名稱
Claude 用來呼叫工具的唯一識別碼
描述
工具做什麼, Claude 讀取以決定何時呼叫
輸入結構描述
TypeScript 用 Zod, Python 用型別字典或 JSON Schema
處理常式
非同步函式, 必須回傳含 content 陣列的物件
// 用 createSdkMcpServer 包裝, 在應用程式行程內執行
// 工具名稱格式, mcp__{server_name}__{tool_name}
// 需列在 allowedTools 中才能不經提示執行

工具結果可回傳文字、圖片、音訊、resource、resource_link 區塊.處理常式若擲出未捕捉的例外會停止代理迴圈, 若捕捉錯誤並回傳 isError: true 則代理迴圈會繼續.

10. Connect to external tools with MCP 連接 MCP servers

Model Context Protocol（MCP）是連接 AI 代理與外部工具、資料來源的開放標準, 讓代理不需撰寫自訂工具實作即可查詢資料庫、整合 Slack 與 GitHub 等 API.

傳輸方式適用情境
stdio文件給的是要執行的本機指令
HTTP / SSE文件給的是 URL
SDK MCP server自行在程式碼中建置工具, 於 SDK 應用程式內直接執行

MCP 工具需要明確權限才能被 Claude 使用, 用 allowedTools 加上萬用字元 mcp__servername__* 可核准伺服器提供的所有工具.

11. Scale to many tools with tool search 用工具搜尋擴展到大量工具

tool search 讓代理能處理數百或數千個工具, 依需求動態發現與載入, 而非把所有工具定義預先載入情境視窗.這解決情境效率（50 個工具可能用掉 1 到 2 萬個 token）與工具選擇準確度（超過 30 到 50 個工具時準確度下降）兩個挑戰.

預設啟用

啟用時, 工具定義保留在情境視窗之外, 代理收到可用工具的摘要, 需要時搜尋並載入最相關的 3 到 5 個工具.可透過 ENABLE_TOOL_SEARCH 環境變數調整, 設為 auto 時依工具定義佔情境視窗的比例決定, auto:N 可自訂百分比閾值.工具少於約 10 個時預先載入全部通常更快.支援除 Haiku 以外的所有 Claude 模型.

12. Agent Skills in the SDK SDK 中的 Agent Skills

Agent Skills 讓 Claude 具備會自主呼叫的專門能力, 打包為含有指示、描述、選用支援資源的 SKILL.md 檔案.與 subagents（可程式化定義）不同, skills 必須建立為檔案系統產物, SDK 沒有提供程式化的 skill 註冊 API.

skill 位置

選擇位置 :
專案 skills
使用者 skills
plugin skills

.claude/skills/, 透過 git 與團隊共享.

~/.claude/skills/, 跨專案的個人 skill.

隨已安裝的 Claude Code plugin 打包.

在 query() 上設定 skills 選項控制哪些 skill 對此 session 可用, 省略時已發現的 skills 會啟用, 傳入 "all" 啟用全部, 傳入名稱清單只啟用特定的, 傳入 [] 全部停用.SKILL.md 中的 allowed-tools 前置資料只在直接使用 CLI 時有效, 透過 SDK 應改用主要的 allowedTools 選項.

13. Subagents in the SDK SDK 中的子代理

subagents 是主代理可以產生的獨立代理實例, 用來處理聚焦的子任務.可用程式化（agents 參數）、以檔案系統為基礎（.claude/agents/）、或內建通用型 subagent 三種方式建立.

優點

情境隔離, 每個 subagent 全新對話中執行

平行化, 獨立子任務同時完成

專門的指示與知識

工具限制降低意外操作風險

AgentDefinition 主要欄位

description（必要）, prompt（必要）

tools, disallowedTools, model

skills, memory, mcpServers

maxTurns, background, effort, permissionMode

subagent 的情境視窗全新開始, 但不是空的, 唯一從父層到 subagent 的管道是 Agent 工具的提示詞字串.

14. Slash Commands in the SDK SDK 中的斜線指令

斜線指令提供以 / 開頭的特殊指令控制 Claude Code session, 透過在提示詞字串中包含即可送出, 就像一般文字一樣.只有不需要互動式終端機即可運作的指令才能透過 SDK 派送.

/compact  # 壓縮對話歷史, 摘要較舊訊息同時保留重要情境
/clear    # 重置對話為空情境

自訂斜線指令定義為特定目錄中的 markdown 檔案, 建議格式是 .claude/skills/<name>/SKILL.md（.claude/commands/ 為舊格式）.支援 $0、$1 等佔位符處理動態引數, ! 前綴執行 bash 指令並納入輸出, @ 前綴引用檔案內容.

15. Plugins in the SDK SDK 中的外掛

plugins 讓你用可跨專案共享的自訂功能擴充 Claude Code.透過 Agent SDK, 可程式化從本機目錄載入 plugins, 為代理 session 加入 skills、agents、hooks、MCP servers.

限制

載入時 type 欄位必須是 "local", 這是 SDK 唯一接受的值.要使用透過 marketplace 或遠端儲存庫散布的 plugin, 需先下載再提供本機目錄路徑, 路徑應指向 plugin 根目錄.

plugin 成功載入後會出現在系統初始化訊息中, 可檢查 message.plugins、message.skills、message.slash_commands 驗證.plugin 提供的 skills 會自動以 plugin 名稱加上命名空間避免衝突, 直接呼叫時送出 /plugin-name:skill-name.

16. Modifying system prompts 修改系統提示詞

系統提示詞定義 Claude 的行為、能力、回應風格.Agent SDK 有三個起始點.

起始點說明 / 適用情境
最小預設值未設定時, 涵蓋工具呼叫但省略 Claude Code 編碼指引
claude_code 預設值CLI 使用的完整系統提示詞, 適合建置類似 Claude Code 的工具
自訂字串你自己撰寫, SDK 只送出你提供的內容, 適合不同介面或權限模型

CLAUDE.md 給 Claude 持久的專案情境, SDK 把其內容注入對話而非系統提示詞, 因此可搭配任何系統提示詞設定運作.output styles 是儲存的設定, 以 markdown 檔案儲存, 可跨 session 與專案重複使用.

18. Intercept and control agent behavior with hooks SDK 中的 hooks

hooks 是在代理事件發生時執行你程式碼的回呼函式, 可在危險操作執行前封鎖它們、記錄並稽核每次工具呼叫、轉換輸入輸出、要求人工核准敏感動作.

PreToolUse工具呼叫請求, 可封鎖或修改
PostToolUse工具執行結果
UserPromptSubmit使用者提示詞送出
Stop代理執行停止
SubagentStart / SubagentStop子代理啟動與結束
PreCompact對話壓縮請求
PermissionRequest權限請求, 部分事件僅 TypeScript SDK 支援

每個 hook 回呼收到輸入資料、工具使用 ID、情境三個引數, hookSpecificOutput 中的 permissionDecision 可設為 allow、deny、ask、defer.hooks 在你的應用程式行程中執行, 不消耗代理的情境視窗.

19. Rewind file changes with checkpointing SDK 中的檔案回退

檔案 checkpointing 追蹤透過 Write、Edit、NotebookEdit 工具在代理 session 期間所做的檔案修改, 讓你能把檔案回退到任何先前狀態.透過 Bash 指令（如 echo > file.txt）做的變更不會被捕捉.

1 · 啟用
2 · 擷取 checkpoint
3 · 回退

在選項中啟用 enable_file_checkpointing=True 並設定 extra_args={"replay-user-messages": None}.

從第一則使用者訊息擷取回應串流中的 checkpoint UUID.

以空提示詞回復該 session 並呼叫 rewindFiles(), 只回復磁碟上的檔案, 不會回退對話本身.

限制

只追蹤 Write/Edit/NotebookEdit 工具, checkpoint 綁定於建立它的 session, 只回退檔案內容不含目錄的建立或刪除, 只支援本機檔案.

20. Get structured output from agents 取得結構化輸出

結構化輸出讓你定義想要從代理取得的資料確切形狀.代理仍可使用任何需要的工具完成任務, 最後取得符合結構描述的驗證 JSON, 不符合時重新提示.

// 透過 outputFormat（TS）或 output_format（Python）傳入 JSON Schema
// 完成後, ResultMessage 包含 structured_output 欄位
// 可用 Zod（TS）或 Pydantic（Python）定義結構描述並產生 JSON Schema
失敗情境

結構描述過於複雜、任務本身模糊不清、或達到重試上限時可能失敗.結果訊息 subtype 為 error_max_structured_output_retries 表示這種失敗, 應檢查此欄位以區分驗證失敗與模型回退撤回輸出.

21. Todo Lists 待辦清單追蹤

待辦清單追蹤提供有結構的方式管理任務並向使用者顯示進度.自 TypeScript Agent SDK 0.3.142 與 Claude Code v2.1.142 起, session 改用結構化的 Task 工具 TaskCreate、TaskUpdate、TaskGet、TaskList 取代 TodoWrite.

pending識別出任務時建立
in_progress開始工作時啟動
completed任務成功結束時完成
移除群組中所有任務都完成時

SDK 會為需要 3 個以上明確動作的複雜多步驟任務、使用者提供的任務清單、非瑣碎操作、使用者明確要求時自動建立待辦.監控程式碼仍檢視助理串流中的 tool_use 區塊, 但改為依任務 ID 維護對照表而非在每次呼叫時取代整個清單.

22. Handle approvals and user input 處理核准與使用者輸入

Claude 在兩種情況下會要求使用者輸入, 需要工具使用權限、以及有澄清性問題時（透過 AskUserQuestion 工具）.兩者都會觸發你的 canUseTool 回呼, 暫停執行直到你回傳回應.

回呼觸發時機

工具需要核准, 未被權限規則或模式自動核准

Claude 提出問題, 呼叫 AskUserQuestion 工具

自動核准的工具永遠不會觸發此回呼

回呼可回傳的決定

允許（PermissionResultAllow）或拒絕（PermissionResultDeny）

修改工具輸入

核准並記住, 回傳建議規則避免下次再詢問

拒絕並建議替代方案, 或完全重新導向

若需要套用到每次工具呼叫的邏輯, 應使用 PreToolUse hook 而非 canUseTool.

23. Track cost and usage 追蹤成本與用量

24. Observability with OpenTelemetry SDK 的可觀測性

25. Securely deploying AI agents 安全部署 AI 代理

Claude Code 與 Agent SDK 能執行程式碼、存取檔案、代表你與外部服務互動, 其行為可能受到處理內容（檔案、網頁、使用者輸入）影響, 有時稱為提示注入.安全部署適用執行任何半信任程式碼的相同原則, 隔離、最小權限、縱深防禦.

隔離技術隔離強度效能負擔
sandbox runtime輕量低
containers設定決定視設定而定
gVisor優異中高
VMs優異高

建議把敏感資源（如憑證）放在包含代理的邊界之外, 例如透過代理外部執行的 proxy 注入 API 金鑰.核心模式是代理透過掛載的 Unix socket 與外部代理程式溝通, 即使代理被入侵也無法任意外洩資料.

26. Hosting the Agent SDK 部署 Agent SDK

Agent SDK 產生並監督一個 claude CLI 子行程, 擁有 shell、工作目錄、以及磁碟上的 session 檔案.每個 session 對應一個子行程, 三種狀態預設存在本機磁碟（session 逐字稿、CLAUDE.md 記憶檔案、工作目錄產物）, 不會在容器重啟或縮減規模時存活.

Session 模式說明
一次性 session為每個使用者任務建立容器並在完成後銷毀
長期執行 session執行持久的容器實例服務持續工作
混合 session一次性容器啟動時從 SessionStore 補水並持久化更新回去
多代理容器一個容器內執行多個 SDK 子行程做緊密協作

正式環境考量包含 session 與狀態持久化、可觀測性（容器或編排器層級設定 OTEL 環境變數）、驗證與密鑰（子行程讀取 ANTHROPIC_API_KEY 或透過 proxy 注入）、擴展與並行（並行數受限於主機 RAM）、多租戶隔離（settingSources: [] 加上每租戶工作目錄與 CLAUDE_CONFIG_DIR）.

30. TypeScript SDK V2 session API 已移除

V2 session API 已不再支援, TypeScript Agent SDK 0.3.142 移除了 unstable_v2_createSession、unstable_v2_resumeSession、unstable_v2_prompt, 以及 SDKSession 與 SDKSessionOptions 型別.

已移除

V2 曾是實驗性的 session API, 免除非同步產生器與 yield 協調的需求, 改用每回合各自的 send()/stream() 循環, API 簡化為三個概念, createSession()/resumeSession() 開始或延續對話、session.send() 送出訊息、session.stream() 取得回應.

要遷移, 應改用 query() API 加上其接受的 session 選項, 傳入 AsyncIterable<SDKUserMessage> 做多輪對話, 或用 options.resume 延續已儲存的 session.這頁文件僅保留供維護 Agent SDK 0.2.x 以下版本程式碼的人參考.
