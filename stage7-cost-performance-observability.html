1. 有效管理成本

Claude Code 依 API token 消耗計費.企業部署中平均每位開發者每個活躍工作日約 13 美元, 每月約 150 到 250 美元, 90% 使用者每日花費維持在 30 美元以下.

切換不相關工作時用 /clear, 用 /compact 加上自訂指示告訴 Claude 要保留什麼重點內容.

Sonnet 處理多數編碼任務且比 Opus 便宜, Opus 保留給複雜架構決策, 簡單 subagent 任務可指定 model: haiku.

優先使用 CLI 工具如 gh、aws、gcloud, 它們不會增加逐工具的清單負擔, 並停用未在使用的 MCP server.

用 hook 過濾冗長的日誌檔案, 只回傳符合條件的行, 用 skill 提供領域知識讓 Claude 不需要逐一探索檔案.

agent teams 會產生多個各自有獨立情境視窗的實例, token 用量隨活躍 teammate 數量增加, 控制成本應維持小規模團隊並在工作完成後結束 teammate.

2. Claude Code 如何使用 prompt caching

沒有快取時, API 每回合都要重新處理完整歷史記錄, 有快取時則重複利用已處理過的部分, 只對變動的部分做新的運算.

快取的三個層級

層級內容何時改變
系統提示詞核心指示、工具定義、輸出風格工具定義集合改變, 或 Claude Code 升級時
專案情境CLAUDE.md、auto memory、無範圍規則session 開始時, 或 /clear、/compact 之後
對話內容你的訊息、Claude 的回應、工具結果每回合

3. 探索情境視窗

官方文件提供一個互動式視覺化模擬, 展示情境視窗在一次 session 中如何逐步填滿, 包含哪些內容自動載入、每次檔案讀取的成本、以及規則與 hooks 何時觸發.

啟動時自動載入的內容

系統提示詞核心行為、工具使用、回應格式指示, 永遠最先載入
Auto memoryClaude 給自己的筆記, 前 200 行或 25KB 以先到者為準
MCP 工具（延遲）只列出工具名稱, 完整結構描述等到需要時才載入
Skill 描述一行式描述, 完整內容只在實際呼叫時才載入
CLAUDE.md全域與專案層級偏好設定, 每個對話開始時載入

壓縮後保留的內容

機制壓縮後狀態
系統提示詞與輸出風格不變, 不屬於訊息歷史

帶 paths 前置資料的規則遺失, 直到再次讀取符合的檔案

4. Monitoring 監控

透過 OpenTelemetry（OTel）匯出遙測資料, 追蹤組織內的 Claude Code 用量、成本、與工具活動.

快速設定

# 1. 啟用遙測
export CLAUDE_CODE_ENABLE_TELEMETRY=1
# 2. 選擇匯出器
export OTEL_METRICS_EXPORTER=otlp
export OTEL_LOGS_EXPORTER=otlp
# 3. 設定 OTLP 端點
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317
claude

可用的指標

claude_code.session.count
CLI session 啟動次數
claude_code.cost.usage
Claude Code session 的成本, 單位 USD
claude_code.token.usage
使用的 token 數量
claude_code.pull_request.count
建立的 pull request 數量
分散式追蹤

需同時設定 CLAUDE_CODE_ENABLE_TELEMETRY=1 與 CLAUDE_CODE_ENHANCED_TELEMETRY_BETA=1, 每個使用者提示詞會開啟一個 claude_code.interaction 根 span.
