2. Common workflows

探索程式碼庫, 修復錯誤, 重構, 測試等日常任務的逐步指南

選擇工作流程 : 理解程式碼庫 修復錯誤 重構程式碼 撰寫測試 建立 PR 處理文件

先問 "give me an overview of this codebase", 再深入 "explain the main architecture patterns used here" 或 "how is authentication handled?". 找相關程式碼時, 用 "find the files that handle user authentication", 再問元件如何互動, 最後追蹤執行流程.

把錯誤訊息分享給 Claude ("I'm seeing an error when I run npm test"), 請它提供修復建議, 再套用修復方案. 告知重現指令, 以及錯誤是間歇性還是持續發生.

先找出過時的程式碼 ("find deprecated API usage"), 取得重構建議, 在維持行為不變的前提下套用變更, 最後執行測試驗證. 建議以小步驟, 可驗證的方式進行重構.

找出未被測試覆蓋的程式碼, 產生測試骨架, 針對邊界情況新增有意義的測試案例, 最後執行並驗證. Claude 會檢視既有測試檔案以符合專案慣用的風格與框架.

可直接請 Claude "create a pr for my changes", 或逐步進行 : 摘要變更, 產生 PR, 審閱並精煉描述. 用 gh pr create 建立的 PR 會自動連結目前 session, 之後可用 claude --from-pr <number> 回來.

找出缺乏文件的程式碼, 產生文件（JSDoc, docstring 等）, 審閱並補充脈絡與範例, 最後確認是否符合專案標準.

3. Commands

Claude Code 內建指令與 bundled skills 的完整參考

依工作階段查看 : 第一次進入專案 任務進行中 平行工作 交付前 session 之間

執行 /init 產生初版 CLAUDE.md, 再用 /memory 精修. 用 /mcp 與 /agents 設定專案需要的伺服器或子代理, 用 /permissions 設定核准規則.

/plan 在大規模變更前切換到 plan mode. /model 與 /effort 調整推理程度. 對話變長時, /context 顯示 context 使用狀況, /compact 進行摘要, /btw 處理不想污染歷史的旁支問題.

/agents 開啟子代理管理介面, /tasks 列出背景執行中的工作. /background 讓整個 session 分離到背景執行並釋放終端機. /batch 把大範圍變更拆分, 在各自的 worktree 中執行.

/diff 顯示變更內容, /code-review 檢查目前差異的正確性錯誤與清理機會（--fix 套用, --comment 發表為 PR 行內留言), /review 審閱 GitHub PR, /security-review 做更深入的安全審查, /code-review ultra 執行雲端多代理審查.

/clear 在保留專案記憶的情況下開始新任務. /resume 與 /branch 回到或分支出先前的對話. /teleport 把 web session 拉進終端機, /remote-control 讓其他裝置能繼續這個本機 session.

常用指令速查

指令用途
/init依專案結構初始化 CLAUDE.md
/permissions管理 allow, ask, deny 權限規則
/goal [condition]設定完成條件, 讓 Claude 持續工作直到達成
/schedule [description]建立, 更新, 列出或執行 routines
/usage顯示 session 成本, 方案用量與活動統計

5. Choose a permission mode

控制 Claude 是否要先詢問才能編輯檔案或執行指令

點選查看說明 : default acceptEdits plan auto dontAsk bypassPermissions
不詢問即可執行僅讀取操作
適合剛開始使用, 敏感工作
不詢問即可執行讀取, 檔案編輯, 常見檔案系統指令（mkdir, touch, mv, cp）
適合迭代你正在審閱的程式碼
不詢問即可執行僅讀取操作, Claude 提出計畫但不修改原始檔案
適合在修改前先探索程式碼庫
不詢問即可執行所有操作, 但有背景分類器安全檢查
適合長時間任務, 減少提示疲勞. 連續被擋 3 次或累計 20 次會暫停並恢復詢問
不詢問即可執行僅已預先核准的工具
適合鎖定的 CI 與腳本環境, 完全非互動
不詢問即可執行所有操作, 完全跳過安全檢查
適合僅限隔離的容器或 VM, 無法從啟動時的旗標之外進入此模式

除了 bypassPermissions 之外, 對受保護路徑（如 .git, .claude, .vscode）的寫入永遠不會被自動核准.

重點

在 CLI 中按 Shift+Tab 可循環切換 default, acceptEdits, plan 模式; auto 與 bypassPermissions 需另外啟用才會加入循環.

6. Configure permissions

用細粒度的權限規則, 模式與管理政策控制 Claude Code 可以做什麼

權限系統三層

工具類型範例需要核准
僅讀取File reads, Grep否
Bash 指令Shell execution是
檔案修改Edit/write files是

/permissions 開啟互動介面. 規則依 deny → ask → allow 順序評估, 第一個符合的規則決定結果.

規則語法範例

萬用字元

Bash(npm run *) 允許

Bash(git push *) 拒絕

Bash(ls *) 符合 ls -la, 但不符合 lsof

路徑與網域規則

Read(./.env) 專案根目錄下的 .env

Read(~/Documents/*.pdf) 家目錄下的 PDF

WebFetch(domain:*.example.com) 任何子網域

設定優先順序

1. Managed settings最高權限, 不可被任何層級覆寫
2. 命令列參數臨時的 session 覆寫
3. Local settings.claude/settings.local.json
4. Project settings.claude/settings.json, 團隊共用
5. User settings~/.claude/settings.json, 最低優先權

只要任一層級拒絕某個工具, 其他層級都無法允許它.

7. Claude Code settings

用全域與專案層級設定, 以及環境變數配置 Claude Code

選擇範圍 : Managed User Project Local
位置Server-managed settings, plist / registry, 或系統層級 managed-settings.json
影響對象組織全體成員或機器上所有使用者
團隊共用是（由 IT 部署）
位置~/.claude/
影響對象你自己, across 所有專案
團隊共用否
位置.claude/ 於 repository 內
影響對象此 repository 的所有協作者
團隊共用是（提交到 git）
位置.claude/settings.local.json
影響對象你自己, 僅此 repository
團隊共用否（自動加入 gitignore）

優先順序 : Managed（最高）> 命令列參數 > Local > Project > User（最低）. 陣列型設定會跨層級合併, 純量設定則採用最具體層級的值.

重點設定項目

autoCompactEnabled
預設 true, 接近 context 上限時自動摘要對話
autoMemoryEnabled
預設 true, 啟用自動記憶
fileCheckpointingEnabled
預設 true, 編輯前快照檔案供 /rewind 使用
fallbackModel
主要模型超載時依序嘗試的備援模型
editorMode
"normal" 或 "vim", 輸入提示的按鍵模式
cleanupPeriodDays
預設 30 天, 超過此天數的 session 檔案於啟動時刪除

用 CLAUDE.md 給予持久指示, 並讓 Claude 用自動記憶累積學習

CLAUDE.md 檔案

由你撰寫

內容為指示與規則

範圍 : 專案, 使用者或組織

用於 : 編碼標準, 工作流程, 專案架構

自動記憶

由 Claude 撰寫

內容為學習與模式

範圍 : 每個 repository, 跨 worktree 共享

用於 : 建置指令, 除錯洞見, Claude 發現的偏好

CLAUDE.md 存放位置

範圍位置共享對象
Managed policy系統層級路徑組織內所有使用者

重點

自動記憶預設開啟, 每個專案的記憶目錄在 ~/.claude/projects/<project>/memory/, MEMORY.md 是索引（session 開始時載入, 上限 200 行或 25KB）, 主題檔案則依需要載入.

      "bindings": {
        "ctrl+e": "chat:externalEditor",
        "ctrl+u": null

}
修飾鍵語法

ctrl / control

shift

alt / opt / option / meta

13. Configure your terminal for Claude Code

修正 Shift+Enter 換行, 取得完成提示音, 設定 tmux, 配色主題與 Vim 模式

選擇你遇到的狀況 : Shift+Enter 直接送出 macOS Option 鍵沒反應 完成時沒有提示音 在 tmux 中使用 畫面閃爍或跳動

按 Ctrl+J 或輸入 \ 再按 Enter 可換行不送出, 任何終端機都適用. VS Code, Cursor, Devin Desktop, Alacritty, Zed 執行一次 /terminal-setup 即可啟用 Shift+Enter.

在 Apple Terminal 開啟 Settings → Profiles → Keyboard, 勾選 "Use Option as Meta Key". 在 iTerm2 則於 Settings → Profiles → Keys → General 把 Option 設為 "Esc+".

預設只有 Ghostty, Kitty, iTerm2 會顯示桌面通知. 其他終端機可設定 preferredNotifChannel 為 "terminal_bell", 或設定 Notification hook 播放自訂音效.

# 加入 ~/.tmux.conf
set -g allow-passthrough on
set -s extended-keys on
set -as terminal-features 'xterm*:extkeys'

這讓通知能傳到外層終端機, 也讓 tmux 能區分 Shift+Enter 與一般 Enter.

執行 /tui fullscreen 切換到全螢幕渲染模式, 它會畫在終端機的替代畫面緩衝區, 記憶體使用量維持固定並支援滑鼠操作.

用 /theme 選擇符合終端機的配色主題, auto 選項會跟隨作業系統的亮 / 暗背景. 也可以在 ~/.claude/themes/ 建立自訂主題 JSON 檔.

14. Fullscreen rendering

啟用更流暢, 無閃爍的渲染模式, 支援滑鼠操作並維持穩定的記憶體使用量

Fullscreen rendering 是研究預覽階段的替代渲染路徑, 消除閃爍, 在長對話中維持記憶體平穩, 並新增滑鼠支援, 繪製於終端機的替代畫面緩衝區（如 vim 或 htop）.

傳統模式

用 Cmd+f 或 tmux 搜尋找文字

用終端機原生點擊拖曳選取並複製

Cmd 點擊開啟 URL

全螢幕模式

Ctrl+o 進入 transcript 模式, 再用 / 搜尋或 [ 寫回 scrollback

應用程式內選取, 放開滑鼠自動複製

macOS 用 Cmd, 其他平台用 Ctrl 點擊開啟連結

捲動快速鍵

PgUp / PgDn向上或向下捲動半個畫面
Ctrl+Home跳到對話開頭
Ctrl+End跳到最新訊息, 重新啟用自動跟隨
滑鼠滾輪每次捲動幾行

執行 /tui fullscreen 啟用, /tui default 切回傳統渲染器. 若要保留原生文字選取功能, 可設定 CLAUDE_CODE_DISABLE_MOUSE=1 停用滑鼠捕捉, 同時仍保有無閃爍渲染的優點.

15. Speed up responses with fast mode

在 Claude Code 中切換 fast mode, 取得更快的 Opus 回應

Fast mode 是 Claude Opus 的高速設定, 最快可達 2.5 倍速度, 但每 token 成本較高. 這不是不同的模型, 只是不同的 API 設定, 優先考量速度而非成本. 僅支援 Opus 4.8 與 Opus 4.7.

成本對照（每 MTok）

Opus 4.8, 輸入 $10 / 輸出 $50

Opus 4.7, 輸入 $30 / 輸出 $150

首次在對話中啟用時, 需為整個既有 context 支付完整的 fast mode 價格, 所以在對話一開始就啟用最划算.

何時使用

適合 : 快速迭代, 即時除錯, 時間緊迫的工作

不適合 : 長時間自主任務, 批次處理, 成本敏感的工作負載

可與較低的 effort level 併用, 在簡單任務上取得最高速度

切換方式 : 輸入 /fast 按 Tab 切換, 或在使用者設定中設 "fastMode": true. 啟用時若模型不是 Opus 會自動切換過去. 速率限制用盡時會自動退回標準速度, 冷卻結束後自動恢復.

16. Model configuration

了解 Claude Code 的模型設定, 包含 opusplan 等模型別名

模型別名

default
清除任何覆寫, 回到你帳號類型建議的模型
sonnet
最新的 Sonnet 模型
opus
最新的 Opus 模型
haiku
最新的 Haiku 模型
fable
最新的 Fable 模型
opusplan
plan mode 用 Opus, 切換到實作階段後改用 Sonnet

用 /model 在 session 中切換（開啟選取器, 按 s 只切換本次 session), 或啟動時用 claude --model <name>.

調整 effort level

Effort 控制 Claude 使用多少自適應推理的思考預算. 用 /effort [level] 設定, 接受 low, medium, high, xhigh, max, 或 ultracode（僅限本次 session, 結合 xhigh 推理與自動工作流程協調）.

備援模型鏈

claude --fallback-model sonnet,haiku

主要模型超載時, Claude Code 會依序切換到鏈中下一個可用模型, 最多可設定三個模型.

重點

管理員可用 availableModels 限制使用者, 子代理, skills 與 advisor 可選擇的模型, 搭配 enforceAvailableModels 讓限制也套用到 Default 選項.
