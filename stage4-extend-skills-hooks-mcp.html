1. Extend Claude Code

理解何時使用 CLAUDE.md, Skills, subagents, hooks, MCP 與 plugins

依目標選擇功能

CLAUDE.md
每次對話都載入的持久性 context, 適合專案慣例與規則
Skill
Claude 可使用的指示知識與工作流程, 依需求載入
Subagent
隔離的執行 context, 只回傳摘要結果
MCP
連接外部服務, 提供資料或動作存取
Hook
在生命週期事件上保證觸發的確定性自動化
Plugin
把以上功能打包成單一可安裝, 可分享的單元

比較容易混淆的功能

選擇比較對象 : Skill vs Subagent CLAUDE.md vs Skill Hook vs Skill MCP vs Skill
Skill

可重複使用的內容, 載入任何 context

適合參考材料與可呼叫的工作流程

Subagent

隔離的工作者, 獨立於主對話執行

適合會讀很多檔案, 需要 context 隔離的任務

CLAUDE.md

每個 session 自動載入完整內容

適合 "一律要做 X" 的規則

Skill

依需求載入, 可用 /<name> 觸發

適合參考資料或可呼叫的工作流程

Hook

保證在事件上觸發, Claude 無法選擇不執行

適合編輯後檢查, 封鎖不安全指令

Skill

Claude 自行判斷如何套用步驟

適合需要推理的多步驟工作流程

MCP

提供連接外部系統的工具與資料存取

例如 Slack 整合, 資料庫查詢, 瀏覽器控制

Skill

提供如何有效使用這些工具的知識

例如資料模型, 常用查詢樣式, 訊息格式規則

重點

Plugin 是封裝層, 把 skills, hooks, subagents 與 MCP 伺服器打包成單一單元, 方便跨專案重複使用或分享給團隊.

4. Automate actions with hooks

在 Claude Code 編輯檔案, 完成任務或需要輸入時自動執行 shell 指令

Hooks 提供確定性控制, 確保某些動作一定會發生, 而非仰賴 LLM 自行選擇是否執行.

常見自動化範例

選擇範例 : 完成通知 編輯後自動格式化 封鎖受保護檔案編輯 壓縮後重新注入 context 自動核准權限提示

用 Notification 事件, 在 Claude 等待輸入或權限時觸發桌面通知（macOS 用 osascript, Linux 用 notify-send).

用 PostToolUse 事件搭配 Edit|Write matcher, 在編輯後自動執行 Prettier 等格式化工具.

用 PreToolUse 事件檢查目標檔案路徑, 若符合 .env, package-lock.json 等保護樣式, 以結束代碼 2 封鎖.

用 SessionStart 事件搭配 compact matcher, 在每次壓縮後重新提醒 Claude 專案慣例與目前工作重點.

用 PermissionRequest 事件, 針對特定工具（如 ExitPlanMode）回傳 JSON "behavior": "allow" 自動核准.

結束代碼行為

結束代碼 0
沒有異議, 動作正常進行（部分事件會把 stdout 加入 context)
結束代碼 2
動作被封鎖, stderr 內容回饋給 Claude 做為理由

5. Hooks reference

Hook 事件, 設定 schema, JSON 輸出格式, 結束代碼與進階功能的完整參考

重要事件

SessionStartSession 開始或恢復時
UserPromptSubmit送出提示詞, Claude 處理前
PreToolUse工具呼叫執行前, 可封鎖
PermissionRequest權限對話框出現時
PostToolUse工具呼叫成功後
SubagentStart / Stop子代理產生 / 結束時
StopClaude 完成回應時
PreCompact / PostCompactContext 壓縮前 / 後
SessionEndSession 結束時

Hook 類型

command
執行 shell 指令
http
將事件資料 POST 到 URL
mcp_tool
呼叫已連接的 MCP 伺服器工具
prompt
單輪 LLM 評估, 回傳是非決策
agent
多輪驗證, 附帶工具存取（實驗性）

PostToolUse 無法撤銷已執行的動作; PermissionRequest 在非互動模式（-p）不會觸發; Stop hook 連續封鎖 8 次會被覆寫, 讓 Claude 恢復正常結束.

7. Connect Claude Code to tools via MCP

學習如何用 Model Context Protocol 連接 Claude Code 到你的工具

選擇安裝方式 : 遠端 HTTP 遠端 SSE（已棄用） 本機 stdio 遠端 WebSocket 
claude mcp add --transport http notion https://mcp.notion.com/mcp

連接雲端服務最建議使用的方式.

claude mcp add --transport sse asana https://mcp.asana.com/sse

SSE 傳輸已棄用, 有 HTTP 可用時應優先選擇 HTTP.

claude mcp add --env AIRTABLE_API_KEY=KEY --transport stdio airtable \
  -- npx -y airtable-mcp-server

適合需要直接系統存取的工具, 如瀏覽器或自訂腳本.

claude mcp add-json events-server \
  '{"type":"ws","url":"wss://mcp.example.com/socket"}'

適合會主動推送事件給 Claude 的伺服器.

範圍優先順序

1. Local最高優先, 僅目前專案對你自己生效
2. Project透過 .mcp.json 與團隊共享
3. User跨你所有的專案
4. PluginPlugin 提供的伺服器
5. claude.ai connectors最低優先, 於 claude.ai 設定的連接器
重點

MCP tool search 預設啟用, 只有工具名稱在 session 開始時載入, 完整 schema 依需求延後載入, 讓新增更多伺服器對 context 的影響降到最低.

8. Output styles

讓 Claude Code 適應軟體工程以外的用途

Output styles 改變 Claude 如何回應（角色, 語氣, 輸出格式), 而非它知道什麼, 這點與 CLAUDE.md 不同.

選擇內建 style : Default Proactive Explanatory Learning
說明既有系統提示, 為高效完成軟體工程任務而設計
說明立即執行, 對常規決策做出合理假設而非暫停等待, 但仍會顯示權限提示
說明在協助完成任務之間提供教育性的 Insights, 幫助理解實作選擇
說明協作式邊做邊學模式, 用 TODO(human) 標記讓你自己實作部分程式碼

執行 /config 選擇 Output style, 變更於 /clear 或新 session 後生效. 自訂 style 用 frontmatter keep-coding-instructions: true 決定是否保留內建的軟體工程指示.

11. Create and distribute a plugin marketplace

建置並代管 plugin marketplace, 跨團隊與社群分發 Claude Code 擴充功能

選擇 Plugin 來源類型 : 相對路徑 github url git-subdir npm

同一儲存庫內的本機目錄, 如 "./my-plugin", 相對於 marketplace 根目錄解析.

GitHub 儲存庫, 可指定 ref（分支或標籤）與 sha（精確 commit）.

任意 git URL（如 GitLab, Bitbucket, 自架伺服器）.

Git 儲存庫內的子目錄, 用稀疏複製只抓取該子目錄, 適合大型 monorepo.

透過 npm install 安裝的套件, 可指定版本與自訂登錄.

版本管理

Claude Code 依序解析版本 : plugin.json 中的 version → marketplace 條目中的 version → git commit SHA. 設定明確版本後, 每次發布都要手動遞增才能推送更新給使用者.

受管理的限制

管理員可用 strictKnownMarketplaces 限制使用者能新增的 marketplace 來源, 空陣列表示完全鎖定.

13. Constrain plugin dependency versions

宣告 plugin 依賴的版本限制, 讓上游 plugin 發布破壞性變更時你的 plugin 仍能運作

{
  "name": "deploy-kit",
  "dependencies": [
    "audit-logger",
    { "name": "secrets-vault", "version": "~2.1.0" }
  ]
}

version 欄位接受 semver 範圍語法, 如 ~2.1.0, ^2.0, >=1.4. 版本限制依據 marketplace 儲存庫上的 git tag 解析, 命名慣例為 {plugin-name}--v{version}.

常見錯誤

錯誤意義
dependency-unsatisfied宣告的依賴未安裝或已停用
range-conflict多個版本需求無法合併
dependency-version-unsatisfied已安裝的依賴版本超出此 plugin 宣告的範圍
no-matching-tag依賴的儲存庫沒有符合範圍的 tag
# 清理不再被任何 plugin 需要的自動安裝依賴
claude plugin prune

14. Recommend your plugin from your CLI

從你的 CLI 送出一行標記, 讓 Claude Code 提示使用者安裝你的官方 plugin

Claude Code 執行 Bash 或 PowerShell 工具時會設定 CLAUDECODE=1. 你的 CLI 偵測到此變數後, 寫入標記到 stderr :

<claude-code-hint v="1" type="plugin" value="example-cli@claude-plugins-official" />

選擇放置點

--help 輸出
Claude 探索陌生 CLI 時常執行 help
未知子指令錯誤
正好是 Claude 對介面感到困惑的時刻
登入或驗證成功
使用者已處於設定情境
首次執行歡迎訊息
自然的入門時刻

Claude Code 永遠不會自動安裝, 使用者必須確認. 每個 plugin 只提示一次, 每個 session 跨所有 CLI 最多顯示一次提示.

15. Recommend plugins for your org

在 marketplace 條目中加入 relevance 區塊, 讓 Claude Code 依使用者工作內容建議 plugin

{
  "name": "terraform-helpers",
  "relevance": {
    "topic": "Terraform",
    "signals": {
      "cli": ["terraform"],
      "filesRead": ["**/*.tf"]
    }
  }
}

訊號欄位參考

cwd
符合工作目錄的 glob 樣式, 唯一能在 session 開始時就符合的訊號
cli
本 session 執行過的 shell 指令名稱
hosts
Bash 指令中出現過的網域主機名稱
filesRead
符合 Claude 讀取過的檔案路徑的 glob 樣式
重點

Marketplace 宣告的建議是選擇性加入的（opt-in), 管理員必須在 managed settings 的 pluginSuggestionMarketplaces 中將 marketplace 加入允許清單, 才會顯示給使用者.
