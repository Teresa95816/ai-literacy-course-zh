1. Overview

Claude Code 是什麼, 以及在哪些介面可以使用

Claude Code 是一套 AI 驅動的程式設計助理, 能讀取你的程式碼庫, 編輯檔案, 執行指令, 並整合你的開發工具. 它理解你整個專案, 能跨多個檔案與工具完成任務.

選擇你的環境

點選切換 : Terminal VS Code Desktop app Web JetBrains

完整功能的 CLI, 直接在終端機中編輯檔案, 執行指令, 管理整個專案.

# macOS, Linux, WSL
curl -fsSL https://claude.ai/install.sh | bash

# 啟動
cd your-project
claude

VS Code 擴充套件提供行內差異比較, @ 提及檔案, 計畫審閱與對話紀錄, 直接顯示在編輯器中. 在 Extensions 面板搜尋 "Claude Code" 安裝, 之後於命令選擇區輸入 "Claude Code" 並選擇 Open in New Tab.

獨立應用程式, 可在 IDE 或終端機之外執行 Claude Code. 可視覺化檢視差異, 並排執行多個 session, 排程重複任務, 並可啟動雲端 session. 提供 macOS (通用版), Windows (x64), Windows ARM64 安裝檔. 需要付費訂閱.

在瀏覽器中執行 Claude Code, 無需本機設定. 適合啟動長時間執行的任務, 之後再回來查看結果, 或處理本機沒有的程式庫. 於 claude.ai/code 開始使用, 桌面瀏覽器與 Claude iOS App 皆可使用.

適用於 IntelliJ IDEA, PyCharm, WebStorm 等 JetBrains IDE 的外掛, 具互動式差異檢視與選取內容分享功能. 從 JetBrains Marketplace 安裝外掛後重啟 IDE, 外掛需要另外安裝 Claude Code CLI.

Step 1, 安裝 Claude Code

使用 Native Install, Homebrew 或 WinGet 其中一種方式安裝.

Step 2, 登入你的帳號

Claude Code 需要帳號才能使用. 執行 claude 後, 第一次使用會提示你登入.

claude
# 之後若要切換帳號或重新驗證

Step 3, 啟動你的第一個 session

cd /path/to/your/project
claude

你會看到 Claude Code 提示字元, 上方顯示版本, 目前模型與工作目錄. 輸入 /help 查看可用指令, 或 /resume 繼續先前的對話.

Step 4, 問你的第一個問題

試試看 : what does this project do? / where is the main entry point? / explain the folder structure. Claude Code 會依需要讀取你的專案檔案, 不必手動附加內容.

Step 5, 進行第一次程式修改

試試 : add a hello world function to the main file. Claude Code 會找到適合的檔案, 顯示修改建議, 徵求你的同意, 再進行編輯. Claude Code 修改檔案前一定會先徵求許可.

Step 6, 用 Git 與 Claude Code 協作

Claude Code 讓 Git 操作變成對話 : what files have I changed? / commit my changes with a descriptive message / help me resolve merge conflicts.

Step 7, 修復錯誤或新增功能

用自然語言描述你想要的, 例如 : there's a bug where users can submit empty forms - fix it. Claude Code 會定位相關程式碼, 理解脈絡, 實作解法, 並執行既有測試.

Step 8, 練習其他常見工作流程

重構程式碼, 撰寫單元測試, 更新文件, 或請 Claude 檢視你的變更並提出改善建議.

常用指令

Shell 指令（在終端機執行）

claude 啟動互動模式

claude "task" 執行一次性任務

claude -p "query" 執行單次查詢後結束

claude -c 繼續目前目錄最近的對話

claude -r 從清單中恢復先前對話

Session 指令（在 Claude Code 內執行）

/clear 清除對話紀錄

/help 顯示可用指令

新手小訣竅

與其說「fix the bug」, 不如說「fix the login bug where users see a blank screen after entering wrong credentials」, 具體的請求能減少來回修正的次數.

3. How Claude Code works

理解代理迴圈, 內建工具, 以及 Claude Code 如何與你的專案互動

代理迴圈（The agentic loop）

當你交付一項任務, Claude 會經歷三個階段, 這些階段會彼此交織進行 :

收集脈絡搜尋檔案, 理解你的程式碼
採取行動編輯程式碼, 進行修改
驗證結果執行測試, 檢查工作成果

每次工具使用回傳的資訊, 都會餵回這個迴圈, 影響 Claude 下一步的決策. 你也是這個迴圈的一部分, 隨時可以打斷 Claude, 引導方向, 提供額外脈絡.

內建工具的五大類別

檔案操作
讀取檔案, 編輯程式碼, 建立新檔案, 重新命名與整理
搜尋
依樣式尋找檔案, 用正規表示式搜尋內容, 探索程式碼庫
執行
執行 shell 指令, 啟動伺服器, 執行測試, 使用 git
網頁
搜尋網頁, 擷取文件, 查詢錯誤訊息
程式碼智慧
編輯後檢視型別錯誤與警告, 跳轉定義, 尋找參照

權限模式（Permission modes）

按 Shift+Tab 可在下列模式間切換 :

點選查看說明 : Default Auto-accept edits Plan mode Auto mode

Claude 在編輯檔案與執行 shell 指令前會先詢問你.

Claude 會編輯檔案並執行常見的檔案系統指令（如 mkdir, mv）而不詢問, 其他指令仍會詢問.

Claude 只探索並提出計畫, 不會修改你的原始檔案, 權限提示規則與 Default 模式相同.

Claude 用背景安全檢查評估所有動作, 目前為研究預覽階段.

重點

Claude Code 是圍繞 Claude 模型的「代理外殼」, 提供工具, 脈絡管理與執行環境, 把語言模型變成能實際動手的程式設計代理.

5. Authentication

登入 Claude Code, 並為個人, 團隊與組織設定驗證方式

個人使用者
  • Claude Pro 或 Max 訂閱, 用 Claude.ai 帳號登入
  • Claude Console, 用 Console 憑證登入
團隊與組織
  • Claude for Teams 或 Enterprise, 集中帳單與團隊管理
  • 雲端供應商 : Amazon Bedrock, Google Vertex AI, Microsoft Foundry
  • 自架的 Claude apps gateway, 以企業 SSO 登入

驗證優先順序

當同時存在多組憑證, Claude Code 依下列順序選擇 :

1. 雲端供應商憑證設定 CLAUDE_CODE_USE_BEDROCK / VERTEX / FOUNDRY 時
2. ANTHROPIC_AUTH_TOKEN以 Bearer token 形式送出
3. ANTHROPIC_API_KEY直接使用 Anthropic API 金鑰
4. apiKeyHelper 腳本適合動態或會輪替的憑證
5. CLAUDE_CODE_OAUTH_TOKEN長期 OAuth token, 適合 CI 流程
6. /login 訂閱憑證Claude Pro, Max, Team, Enterprise 的預設方式

已登入的 Claude apps gateway session 會優先於以上所有項目.

產生長期 token

適用於無法互動登入的 CI 流程或腳本 :

claude setup-token
# 將輸出的 token 設為環境變數
export CLAUDE_CODE_OAUTH_TOKEN=your-token

作業系統 : macOS 13.0+, Windows 10 1809+ 或 Windows Server 2019+, Ubuntu 20.04+, Debian 10+, Alpine Linux 3.19+.

硬體 : 4 GB 以上記憶體, x64 或 ARM64 處理器. 需要網路連線, 支援 Bash, Zsh, PowerShell 或 CMD.

可用 Native Install, Homebrew, WinGet, 或在 Debian / Fedora / RHEL / Alpine 用 apt, dnf, apk 安裝. 在 Windows 上, 建議安裝 Git for Windows 以啟用 Bash 工具, 否則會改用 PowerShell 工具.

# 安裝特定版本
curl -fsSL https://claude.ai/install.sh | bash -s 2.1.89
claude --version
claude doctor

claude doctor 會顯示安裝與設定的詳細檢查結果, 也會列出目前最近一次自動更新的結果.

Native 安裝會在背景自動更新. Homebrew, WinGet 與 Linux 套件管理員安裝預設不會自動更新, 需手動執行對應的升級指令.

用 autoUpdatesChannel 設定控制頻道 : "latest"（預設, 即時取得新功能）或 "stable"（約一週前, 略過重大回歸的版本）.

依安裝方式移除對應的執行檔與版本檔案. 移除設定檔（~/.claude, ~/.claude.json）會刪除所有設定, 允許的工具清單, MCP server 設定與 session 紀錄, 請謹慎操作.

9. Best practices for Claude Code

從設定環境到擴大並行 session 規模的實用技巧

Claude Code 是一套代理式編碼環境, 大部分最佳實務都源自同一個限制 : Claude 的 context window 會很快填滿, 而效能會隨著填滿程度下降.

選擇主題 : 驗證你的成果 先探索再規劃 提供具體脈絡 設定你的環境 管理你的 session 自動化與擴大規模

給 Claude 一個能實際執行的檢查方式 : 測試, 建置, 或截圖比對. 沒有可執行的檢查, 「看起來完成了」就是唯一的信號, 你就會變成驗證迴圈本身.

之前

"implement a function that validates email addresses"

之後

"write a validateEmail function. example test cases: ... run the tests after implementing"

讓 Claude 直接寫程式可能會解決錯誤的問題. 建議流程分四階段 : 探索（進入 plan mode 只讀不改）, 規劃（請 Claude 建立詳細實作計畫）, 實作（切出 plan mode 並依計畫驗證）, 提交（commit 並開 PR）.

若你能用一句話描述整個差異, 就可以跳過規劃階段.

Claude 能推斷意圖, 但無法讀心. 具體指出檔案, 限制條件, 與範例樣式.

之前

"fix the login bug"

之後

"users report that login fails after session timeout. check the auth flow in src/auth/..."

撰寫有效的 CLAUDE.md（執行 /init 產生初稿）, 設定權限（auto mode, 允許清單, sandboxing）, 使用 CLI 工具（gh, aws, gcloud）, 連接 MCP servers, 設定 hooks, 建立 skills 與自訂 subagents.

及早且經常地修正方向（Esc 中斷, /rewind 回溯, /clear 重設）, 積極管理 context（/compact, /btw）, 用 subagent 進行調查, 用 checkpoint 回溯, 並替 session 命名以便日後恢復.

用 claude -p "prompt" 執行非互動模式, 用 worktrees, Desktop app 或 agent teams 執行多個並行 session, 針對大型遷移可將任務拆分後用迴圈呼叫 claude -p 逐一處理, 並在任務視為完成前, 讓 subagent 以全新脈絡進行對抗性審閱.

重點

這份指南中的模式不是鐵律, 而是普遍有效的起點. 留意什麼方法有效, 隨時間培養出屬於自己的直覺.
