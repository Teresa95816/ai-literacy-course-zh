什麼是 Claude Code

Claude Code 是 Anthropic 推出的官方指令列(CLI)工具. 使用者可以用自然語言下達任務指令, 由 Claude 直接在使用者的電腦上讀取、建立、修改檔案並執行操作, 完成多步驟任務, 而不只是在對話框中給出文字建議.

指令列(CLI)
在終端機(terminal)中輸入指令, 適合熟悉命令列環境的使用者
桌面應用程式
Mac, Windows 桌面版, 提供圖形介面, 不需要開啟終端機
網頁版
透過 claude.ai/code 在瀏覽器中使用, 不需要安裝任何軟體
IDE 擴充套件
VS Code, JetBrains 擴充套件, 直接在編輯器內使用

核心能力: Claude Code 直接在使用者的工作環境中執行任務. 不是給建議讓你自己去做, 而是幫你把事情做完.

與一般聊天機器人的差異

最關鍵的差異在於「誰來執行」. 一般聊天機器人告訴你怎麼做, Claude Code 直接幫你做.

一般聊天機器人

主要在對話框內回答問題或產生文字內容

使用者仍需要自己動手把結果貼到檔案或系統中

無法直接操作本機檔案或執行指令

每次互動結束後, 環境不會有任何改變

Claude Code

直接操作電腦上的檔案與工具

可建立 Markdown 文件, 執行版本控制指令

可搜尋網路資料並把結果整合到工作環境

每次任務結束後, 工作環境中留有實際成果

安全與權限機制

Claude Code 在執行有風險或難以回復的操作前, 會先說明意圖並請求使用者確認, 而不是自行默默執行.

刪除檔案
刪除操作一旦執行難以回復, 會先說明將刪除哪些內容再請求確認
覆寫內容
覆蓋現有檔案內容前會提示, 讓使用者確認目標檔案正確無誤
對外發送訊息
向外部服務(例如 GitHub, 郵件)送出資料前會先請求確認
大範圍批次異動
影響多個檔案的操作會在執行前清楚說明影響範圍

這個機制讓使用者可以隨時掌控 Claude Code 實際會做的事情. 這也是它與一般自動化腳本最大的差異: 腳本默默執行, Claude Code 先問再動.

基本互動方式

使用方式類似在對話框中打字, 但情境是在終端機或應用程式介面中進行. 一次典型的互動流程如下:

1 · 輸入任務
2 · Claude 說明意圖
3 · 確認請求
4 · 回報結果

使用者在終端機輸入任務描述

用自然語言說明要完成的任務. 任務描述越具體, 結果越準確. 例如指明檔案名稱、期望的格式或提供範例, 會比籠統的描述更有效率.

範例: 「請把 meeting-notes.md 整理成條列式摘要, 格式參照 sop-template.md」比「整理我的會議記錄」更好.

Claude 說明接下來打算做的事

收到任務後, Claude Code 會先說明計劃執行哪些步驟, 讓使用者了解它的意圖, 而不是直接開始動作.

必要時提出確認請求

涉及有風險操作(刪除、覆寫、對外發送)時, 會暫停並等待使用者核准. 使用者可以確認、修改指令, 或取消操作.

完成後回報結果

任務完成後會回報執行結果, 說明做了哪些事、產生了哪些檔案, 或遇到哪些問題. 使用者可以據此繼續下一個任務.

步驟 1 / 4

內建說明與回饋管道

遇到問題或想了解目前環境支援哪些功能, 有兩個主要管道:

/help
在任何介面中輸入 /help 可以取得使用 Claude Code 的相關說明, 以及目前環境支援的指令列表
GitHub Issues
遇到問題或想提供改進建議, 可以到 GitHub 的 Issues 頁面回報
# 回報問題或提供建議的官方管道
https://github.com/anthropics/claude-code/issues

可擴充的指令, Skills

Claude Code 支援以 / 開頭的指令來呼叫特定功能, 這些功能稱為 Skills. 實際提供哪些 Skills 會依安裝的版本與外掛而有所不同.

呼叫方式
以 / 開頭輸入指令名稱, 例如 /help, /review
功能範疇
例如初始化專案說明文件、針對程式碼或文件變更進行審閱等
可用項目
依安裝的版本與所安裝的外掛而有所不同, 不同環境支援的 Skills 清單不盡相同
如何確認
使用前先用 /help 確認目前環境支援哪些項目
版本異動

由於 Claude Code 持續更新, 可用的 Skills 清單會隨版本演進. 本文件只說明機制, 具體支援哪些 Skills 請以當下環境的 /help 輸出為準.

職場工作者的應用情境

四種在辦公環境中最常見的 Claude Code 使用場景:

1 · 建立與維護文件
2 · 重複性檔案整理
3 · 串接版本控制
4 · 簡報草稿產出

建立與維護文件

可以直接請 Claude Code 協助建立、修改, 或統一格式化 Markdown 文件, 例如會議記錄、SOP、教學講義.

實際範例

本系列的 Markdown, Marp, Git & GitHub 教學文件, 就是用這種方式協助產出的.

重複性檔案整理

針對大量檔案的重新命名、分類、格式轉換等重複性工作, 可以用自然語言描述需求, 讓 Claude Code 協助執行.

在每一步給出確認請求, 減少手動操作出錯的風險. 適合需要批次處理但不想寫腳本的場合.

串接版本控制

搭配 Git, 可以請 Claude Code 協助建立分支、整理 commit 訊息, 或檢查目前的修改內容.

讓文件與專案的修改歷程更清楚, 適合想用 Git 但對指令不熟悉的使用者.

簡報草稿產出

搭配 Marp, 可以請 Claude Code 協助撰寫簡報用的 Markdown 草稿, 再自行微調內容與版面.

適合需要快速把會議重點或專案說明轉換成投影片格式的場合.

情境 1 / 4

使用時的基本注意事項

開始使用前, 請留意以下五點:

01
涉及刪除、覆寫, 或對外發送訊息等有風險的操作, 務必先確認內容無誤再核准執行.
02
處理公司機密或個人資料時, 應遵循所屬企業的資安與保密規範.
03
大量檔案異動前, 建議先用 Git 做版本控制, 以便需要時可以回復.
04
任務描述越具體, 結果越精準. 避免使用過於籠統或模糊的指令.
05
功能與指令會持續更新, 遇到本文件未提及的項目, 以 /help 或官方文件為準.

結語

這份文件聚焦在多數職場工作者都能立即理解與使用的核心概念, 包括基本互動方式、安全機制, 以及幾種常見的辦公應用情境. 建議按以下節奏入門:

第一步
先從簡單且風險低的任務開始練習, 例如整理或建立文件, 感受 Claude Code 的操作節奏
第二步
熟悉確認機制後, 嘗試需要多步驟的任務, 例如搭配 Git 管理修改歷程
第三步
逐步嘗試更複雜的工作流程, 例如批次整理檔案或自動化重複性作業

Claude Code 的核心價值在於讓你用「說話」的方式完成原本需要手動操作的任務. 從低風險的任務開始, 建立對它的信任感, 再逐步擴大使用範圍.
